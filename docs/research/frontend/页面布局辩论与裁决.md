# Finance-God 交易页面布局辩论与裁决（历史研究）

> 历史研究记录：本文保留正反辩论过程。其早期 80 px 左侧导航、深色或移动端结论已被
> [前端统一设计规范](../../page-design/01_前端统一设计规范.md) 取代；当前执行顶部导航、
> 浅色可配置桌面工作区，最低宽度 1024 px。

## 文档定位

本文记录三名 subagents 围绕交易页面布局进行的两轮正反辩论和中立裁决。其目标不是选择“信息多”或“信息少”，而是决定哪些关系必须同时可见、哪些状态必须隔离、哪些细节适合渐进披露。

当前页面范围见 [前端设计文档索引](../../page-design/00_前端设计文档索引.md)。

## 辩论角色

| 角色 | 立场 | 核心主张 |
| --- | --- | --- |
| 正方 | 高密度、一体化交易驾驶舱 | 交易是关系判断；行情、计划、订单、组合后果和风控同屏能减少记忆与版本核对错误 |
| 反方 | 任务分层、组合优先、渐进披露 | P0 用户不是专业交易者；页面应映射业务状态机，避免建议、草稿、校验和执行事实同时争夺注意力 |
| 裁判 | 中立评审 | 按决策质量、安全、认知负担、效率、上下文、响应式和研发复杂度评分，并执行红线否决 |

## 共同事实

双方同意以下前提：

1. P0 用户是普通理财用户、长期投资新手和 ETF/基金自主投资者，不是日内高频交易者。
2. 产品主线是目标与授权 → 组合偏离 → 交易计划 → 订单草稿 → 最终复核 → 执行 → 复盘。
3. AI 建议、确定性风控和执行事实必须分层。
4. T03 AI 辅助交易台需要保持单标的行情、研究、草稿和当前订单的上下文连续性。
5. T06 最终复核必须与编辑环境隔离。
6. 页面切换、Tab、抽屉和跨端不能复制 `TradePlan`、`OrderDraft`、`RiskCheckResult`、`Order` 或 `Fill`。
7. 仿真/实盘、账户、数据时点、过期、阻断和状态未知不能被渐进披露隐藏。

## 第一轮陈词摘要

### 正方论点

- 对交易参数的任何修改都会影响费用、现金、组合权重、计划偏离和风控，关系信息应同屏联动。
- 对象 ID 与版本只能保证系统读到同一事实，不能保证用户看见并理解了关键上下文。
- 高密度不等于自由 Widget；可以使用固定网格、只读 Inspector 和一个主操作。
- 不可逆操作进入覆盖层，编辑阶段保持上下文连续。

### 反方论点

- 页面布局应映射对象成熟度，避免 AI 结论、草稿、校验和订单事实因空间邻接被误认成同一状态。
- P0 成功指标是理解、风控和复盘，不是下单速度。
- 关系连续性应由版本化对象和稳定摘要保证，不需要多个完整工作区同时可操作。
- 只有 T03 和 T07 天然需要较高密度，其余页面应先回答一个主问题。

## 第二轮交叉反驳后的收敛

双方最终共同接受：

- 统一 12 列网格、固定全局事实条；
- T00 使用“编辑器 + 只读影响检查器”；
- T01 使用总览与收件箱双栏；
- T02 使用“我的自选/系统候选”明确 Tab 与持久 Inspector；
- T04 使用顶部计划原因和下方动作/组合影响双栏；
- T06 使用最终复核覆盖层；
- T07 使用全宽表格与选中后 Inspector；
- T10 普通模式为证据抽屉，高级模式为独立页；
- M01 使用独立移动单列任务流。

争议集中在 T03、T08 和 T09：

| 页面 | 正方 | 反方 |
| --- | --- | --- |
| T03 | 结构化 AI 策略摘要与订单草稿应持续同屏 | 完整 AI 与完整订单不能同时可操作，只保留只读计划摘要 |
| T08 | 持仓、偏离和数据质量应持续交叉参照 | 避免三窄列和多滚动轴，使用主表与渐进详情 |
| T09 | 原计划、用户修改、实际执行和结果应同屏比较 | 必须先评价过程，结果过早出现会产生结果偏见 |

## 裁决评分

评分维度为决策质量 20%、风险与误操作 18%、认知负担 16%、执行效率 12%、跨页上下文 10%、可验证性 8%、响应式 8%、研发复杂度 8%。

| 页面 | 正方 | 反方 | 裁决 |
| --- | ---: | ---: | --- |
| T03 | 84.2 | 87.6 | 反方小胜，补回正方的上下文连续性 |
| T08 | 86.1 | 85.4 | 正方极小胜，修复窄栏和滚动轴 |
| T09 | 86.5 | 88.9 | 反方小胜，但双方原案均未通过结果顺序要求 |
| 三页均值 | 85.6 | 87.3 | 差距 1.7，未形成压倒性胜出 |

裁判最终选择第三种混合布局，预计 T03、T08、T09 分别达到 92.4、91.9、93.1。

## 最终布局裁决

### T03：固定计划锚点 + 互斥工作态

采用 8/4：

- 左侧：行情、图表、标的事实，以及当前标的持仓/委托/成交 Dock；
- 右侧顶部：112–120 px 只读计划锚点；
- 右侧下部：`研究判断` 与 `订单草稿` 两个互斥工作态；
- T06：独立最终复核覆盖层。

计划锚点永久显示：

- 结构化结论；
- 适用周期；
- `TradePlan` 版本；
- 研究与行情时点；
- 当前草稿相对计划的偏离；
- 过期和阻断状态。

计划锚点不提供主操作。研究态唯一主操作是“创建/打开订单草稿”；订单态唯一主操作是“进入最终复核”。

### T08：主表 + 单一持久检查器

采用：

- 顶部：全宽组合摘要；
- 中部 7/5：当前/目标/计划后配置对比；风险、现金和集中度；
- 底部 8/4：完整持仓主表；单一持久检查器。

检查器永久显示：

- 最大偏离；
- 硬边界状态；
- 数据完整度；
- 数据时点；
- 当前选中对象；
- 对应偏离解释。

持仓、配置区段和偏离 Chip 共用一个选中状态。禁止创建持仓、偏离、数据质量三套独立滚动区。

### T09：过程在前的证据阶梯

页面从上到下按以下顺序：

1. 复盘对象、周期、状态和证据完整度；
2. 原计划/用户修改/实际执行字段级比较；
3. 授权、风控和失效条件；
4. 执行质量与事件时间线；
5. 组合偏离与风险变化；
6. 匹配周期后的结果；
7. 用户反馈与下一步。

结果不能成为首屏英雄数字。计划周期不足时，不展示确定性收益结论。

## 永久可见与渐进披露

### 永久可见

- 仿真/实盘、账户；
- 市场或连接状态；
- 数据时点和延迟；
- Agent 暂停状态；
- 当前对象 ID 与版本；
- 影响主操作的过期、数据完整度和硬阻断；
- 当前页面唯一主操作；
- T03 计划锚点；
- T08 硬边界、数据完整度和当前选中对象；
- T09 当前评价阶段和证据完整度。

### 可渐进披露

- 完整 AI 证据与 Agent 过程；
- 原始规则输入；
- 长历史和完整事件原文；
- 次级图表指标；
- 非阻断的数据质量明细；
- 压力情景参数；
- 内部 Trace 和错误栈。

渐进披露不得隐藏会改变交易决定的账户后果、时点、硬阻断、未确认警告和订单状态。

## 1024 px 退化规则

- 保留 80 px 导航和 48 px 事实条；
- 内容改为 8 列、12 px 间距；
- 模式与账户始终显示；市场、数据和 Agent 可收纳到带最严重状态徽标的入口；
- T03 主区全宽，计划锚点压缩为 64–72 px 粘性条，研究或订单作为 360 px 侧页或全宽工作态；
- T08 配置和风险纵向排列，持仓全宽，检查器作为 360 px 侧页；
- T09 按既定顺序单列，比较矩阵保持可读，不缩小字号；
- 表格详情采用侧页，不压窄主表。

## 390 px 退化规则

- 使用 M01 单列任务流，不同比缩小桌面布局；
- 顶部至少保留模式、账户简称、最严重状态和数据时点；
- T03 只保留标的摘要、简化图表和研究结论；订单进入全屏步骤；
- T08 按摘要 → 配置 → 风险 → 持仓 → 选中项底部页排列；
- T09 按过程 → 执行 → 组合 → 结果 → 反馈排列；
- T06 全屏，底部确认按钮粘性，阻断时移除按钮；
- 表格改为语义列表或键值行；
- 关键状态不能只藏在横向滑动 Tab 中。

## 布局不变量

1. 每个页面只有一个视觉主操作。
2. AI 建议、草稿、复核和执行事实不得共用同一动作层级。
3. T03 永久保留只读计划锚点，完整研究态与订单编辑态互斥。
4. T08 只有一个跨持仓、偏离和数据质量的选中对象与检查器。
5. T09 的结果位于计划一致性和执行质量之后。
6. 阻断、过期、状态未知和弱网在当前任务区持续显示。
7. 一个页面不得出现三个独立纵向滚动轴。
8. Tab、抽屉、路由和跨端只引用同一对象 ID 与版本。
9. 渐进披露不能隐藏模式、账户后果、时点、阻断、警告确认和订单状态。
10. 1024/390 通过重排、侧页和语义列表退化，不通过缩小字体和删减风险信息。

## 待验证假设

| 页面 | 测试任务 | 主要指标 |
| --- | --- | --- |
| T03 | 改数量、切账户/方向、策略过期 | 计划偏离漏读、版本识别、完成时间 |
| T08 | 从数据冲突定位持仓并决定是否生成计划 | 选中丢失、滚动次数、阻断理解 |
| T09 | 在有盈亏结果时先识别授权/计划/执行问题 | 结果偏见、字段矩阵理解 |
| 1024 | 侧页打开后完成原任务 | 上下文丢失、返回恢复 |
| 390 | 弱网、过期、状态未知下继续任务 | 重复提交、模式识别、中断恢复 |

样本至少覆盖 5–8 名长期投资新手与 5–8 名自主投资者。出现硬阻断仍尝试提交、状态未知重下或仿真/实盘误认时，方案直接失败。
