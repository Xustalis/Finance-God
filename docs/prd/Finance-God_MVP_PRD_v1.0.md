## 1. 文档目的与产品决策

### 1.1 文档目的

本 PRD 定义 Finance-God MVP 的目标用户、产品边界、功能需求、业务规则、验收标准与上线依赖，供产品、设计、前后端、数据、AI 与合规团队共同执行。

本文件是首版需求基线，不构成投资、证券、税务或法律建议，也不授权交易执行、资金接入或受监管的投顾活动。任何具体市场、辖区和产品范围，均须在上线前由产品与合规共同确认。

### 1.2 一句话定位

**Finance-God 是一款以证据和长期计划为核心的个人投资研究与组合复盘助手，帮助用户先理解自身约束，再审视持仓和候选资产。**

核心顺序固定为：明确目标与约束 → 形成长期计划 → 诊断当前持仓 → 研究候选资产 → 分析组合影响 → 定期复盘。

### 1.3 MVP 必须达成的结果

1. 用户可在约 10 分钟内完成基本建档并保存一份个人投资计划摘要。
2. 用户可通过手动录入或 CSV 导入持仓，清楚了解资产类别、地区、行业、币种、集中度与目标偏离。
3. 用户研究受支持资产时，可看到可追溯事实、来源、正反观点、风险、未知项与结论失效条件。
4. 系统明确区分“研究资料是否充分”和“是否符合当前计划”，不把研究结论包装成买卖指令。
5. 当资料、证据、资产可得性或风险边界不满足时，系统必须明确显示阻断原因，而不是给出看似完整的结论。

### 1.4 MVP 关键产品决策

| 决策 | MVP 定义 | 原因 |
|---|---|---|
| 决策单元 | 以长期目标、资产类别与整体组合为优先，单一资产仅作为研究对象。 | 降低热点、短期价格和单一标的对决策的干扰。 |
| 资产范围 | 公开 MVP 仅覆盖经主数据确认的 ETF 与公募基金；个股研究仅作为封闭测试能力。 | 先控制主数据、研究质量和合规复杂度。 |
| 交易边界 | 不接入券商、不获取交易密码、不下单、不自动调仓。 | 避免把研究工具扩展为交易或资金产品。 |
| 结论生成 | 目标配置和组合适配由版本化规则引擎生成；AI 只负责证据整理、解释和缺项提示。 | 保证结果可重放、可解释、可审计。 |
| 可用范围 | 仅展示已配置且可验证的市场与产品目录；未启用范围只显示通用资产类别说明。 | 不把“可浏览”误导为“可投资”或“可购买”。 |
| 内容表达 | 禁止买入、卖出、加仓、目标价、收益承诺和制造紧迫感的措辞。 | 保持长期决策辅助定位。 |

## 2. 用户、场景与非目标

### 2.1 目标用户

| 用户 | 典型特征 | 核心任务 | MVP 价值 |
|---|---|---|---|
| 初阶长期投资者 | 有稳定储蓄，但缺少投资框架，容易受市场波动影响。 | 明确应急资金、长期目标和可承受风险。 | 得到一份可理解、可修改、可复盘的长期计划。 |
| 自主投资者 | 已持有基金、ETF 或零散证券。 | 看懂整体风险与重叠，而非逐项关注涨跌。 | 获得持仓暴露、集中度和偏离诊断。 |
| 研究型投资者 | 会主动阅读基金资料和市场信息。 | 判断候选资产是否值得继续跟踪。 | 以来源和反方观点组织研究，不把观点当事实。 |

### 2.2 核心用户故事

- 作为刚开始长期投资的用户，我希望先划出近期用钱和长期资金的边界，避免将应急资金暴露在高波动资产中。
- 作为已有持仓的用户，我希望导入资产后看到重叠、集中度和配置偏离，而不是只看盈亏。
- 作为研究基金或 ETF 的用户，我希望知道它持有什么、风险来自哪里、有哪些反方观点，以及它会如何改变我的既有暴露。
- 作为长期投资者，我希望在市场波动时看到原始计划和需要复盘的事项，而不是收到推动交易的通知。

### 2.3 非目标与使用限制

MVP 不服务日内交易、高频交易、杠杆、期权、加密资产、私募、结构性产品、价格预测、收益预测、税务筹划、法律意见或自动交易需求。

系统不提供个性化的买卖指令、仓位建议、资金划转或账户密码采集。用户不满足年龄、地区、产品可得性或其他准入条件时，不能访问对应的受限产品信息。

## 3. 范围、版本与成功指标

### 3.1 P0 范围

| 能力域 | P0 交付 |
|---|---|
| 投资建档 | 目标、期限、本位币、流动性、应急资金、主要负债、经验、风险偏好和限制条件的引导式录入。 |
| 长期计划 | 基于资料完整度与版本化配置模板生成计划摘要、风险预算、现金边界、目标配置区间和复盘规则。 |
| 持仓诊断 | 手动录入与 CSV 导入；资产标准化、未解析项、暴露、集中度、重叠和偏离分析。 |
| 资产研究 | 受支持基金与 ETF 的研究备忘录，包含来源、事实、推断、支持与反方观点、风险、未知项和失效条件。 |
| 组合影响 | 规则化判断候选资产是否可进入观察清单，并解释可能造成的暴露变化与阻断原因。 |
| 复盘与信任 | 月度复盘、结论过期提醒、版本历史、导出、编辑和删除申请。 |

### 3.2 明确不在 P0

- 券商账户连接、账户密码抓取、交易执行、自动调仓或资金划转；
- 盘中行情、短线信号、价格目标、收益预测和交易提醒；
- 复杂衍生品、加密资产、私募和结构性产品；
- 未经配置和验证的具体产品映射；
- 税务、法律、保险、遗产或信托规划；
- 为用户自动生成或替代其作出的投资决策。

### 3.3 P1 候选范围

在 Beta 验证、数据授权和合规评估完成后，可评估授权账户聚合、多市场产品目录、多语言、目标达成情景模拟、人工顾问协作和交易工作流集成。

### 3.4 成功指标

| 指标 | MVP 目标 | 定义 |
|---|---:|---|
| 建档完成率 | 不低于 55% | 开始建档至达到计划最低资料门槛的用户比例。 |
| 计划保存率 | 不低于 40% | 已生成计划的用户中保存首个版本的比例。 |
| 持仓诊断查看率 | 不低于 50% | 保存持仓快照后查看至少一个诊断模块的比例。 |
| 完整研究阅读率 | 不低于 50% | 打开备忘录后同时查看证据、反方观点和风险区块的比例。 |
| 适配理解度 | 不低于 70% | 任务后能正确区分“研究充分”与“符合当前计划”的用户比例。 |
| 阻断正确率 | 100% | 未满足资料、证据、范围或风险门槛时，不产生正向观察清单结论。 |
| 30 日复盘率 | 不低于 25% | 保存计划的用户中，30 天内完成一次复盘的比例。 |

## 4. 产品体验与信息架构

### 4.1 主流程

1. 用户选择本位币与可用地区，阅读产品边界。
2. 用户完成投资建档，系统显示资料完整度。
3. 规则引擎生成可保存的长期计划摘要。
4. 用户录入或导入当前持仓，确认解析结果。
5. 系统展示持仓暴露、重叠、集中度与目标偏离。
6. 用户研究候选资产，阅读证据、反方观点与风险。
7. 系统在冻结的输入版本上运行组合影响评估，并给出观察清单结论或阻断原因。
8. 用户按月复盘计划、持仓和研究时效。

### 4.2 信息架构

| 一级模块 | 二级页面 | 用户目的 |
|---|---|---|
| 首页 | 今日概览、待办、资料状态 | 了解计划是否仍有效及下一步。 |
| 我的计划 | 投资画像、长期计划、版本历史 | 建立并维护长期目标和边界。 |
| 我的持仓 | 手动录入、CSV 导入、持仓快照、诊断 | 理解现有资产的整体暴露。 |
| 研究 | 资产搜索、研究备忘录、观察清单 | 获取可验证的研究信息。 |
| 组合影响 | 目标偏离、候选资产影响、规则原因 | 判断候选资产和计划的关系。 |
| 信任与设置 | 数据来源、提醒、导出、删除 | 管理数据、权限和产品控制。 |

### 4.3 核心页面要求

| 页面 | 必须呈现 | 禁止呈现 |
|---|---|---|
| 欢迎与范围说明 | 地区、本位币、产品范围、数据与风险披露。 | 未启用范围内的具体产品。 |
| 投资建档 | 进度、可跳过项、资料完整度、缺项后果。 | 将敏感问题伪装成无条件必填。 |
| 我的计划 | 目标、期限、现金边界、风险预算、目标配置、版本和理由。 | 单一资产交易号召。 |
| 当前持仓 | 原始输入、解析状态、覆盖率、暴露、重叠、偏离。 | 静默丢弃或替换未识别资产。 |
| 研究备忘录 | 来源、数据时点、事实与推断、支持与反方、风险和失效条件。 | 无来源的确定性结论。 |
| 组合影响 | 输入版本、评估状态、暴露变化、规则原因、下一步。 | 买入、卖出、加仓或绕过限制的路径。 |
| 月度复盘 | 资料变更、偏离、研究过期、下次复盘日。 | 利用市场波动制造紧迫感。 |

## 5. 详细功能需求

### 5.1 投资建档与范围控制

用户先选择居住地或使用地区、本位币和至少一个目标；再填写目标日期、可投资金额或区间、近期现金需求、应急资金、主要负债、投资经验、风险承受能力、风险偏好与限制条件。系统区分客观的风险承受能力和主观风险偏好。

| ID | 需求 | 优先级 | 验收标准 |
|---|---|---|---|
| PF-01 | 系统要求用户选择本位币和至少一个投资目标。 | P0 | 缺少任一项时，不能生成个人化长期计划。 |
| PF-02 | 系统采集目标日期、现金需求、应急资金、主要负债、经验和风险信息。 | P0 | 每个字段都有已填、区间或未知状态。 |
| PF-03 | 系统分离风险承受能力和风险偏好。 | P0 | 结果页可解释两者含义及不一致情况。 |
| PF-04 | 系统显示资料完整度及其对结果的影响。 | P0 | 缺关键资料时，结果只能标为通用示例。 |
| PF-05 | 用户可编辑画像；影响计划的改动使旧计划失效。 | P0 | 保存变更后，相关计划状态变为需要重新评估。 |
| PF-06 | 系统按配置范围控制具体产品展示。 | P0 | 未启用范围只显示通用资产类别与教育内容。 |

### 5.2 长期计划与目标组合

P0 使用五档版本化配置模板：保守保全、稳健积累、平衡增长、长期增长和高风险增长。模板是目标配置区间的唯一权重来源；规则引擎根据资料决定可用模板，AI 只能解释规则结果、总结风险和追问缺项。

| 模板 | 权益类区间 | 固收类区间 | 现金或短期类区间 | 示例适用条件 |
|---|---:|---:|---:|---|
| 保守保全 | 0–20% | 40–60% | 25–50% | 期限短、流动性要求高或风险能力较低。 |
| 稳健积累 | 20–40% | 40–60% | 10–25% | 中短期限，重视回撤控制。 |
| 平衡增长 | 40–60% | 25–45% | 5–15% | 中长期目标，现金流稳定。 |
| 长期增长 | 60–80% | 15–35% | 0–10% | 期限长，风险承受能力较高。 |
| 高风险增长 | 80–95% | 5–20% | 0–5% | 期限很长，财务韧性较强且理解较大波动。 |

| ID | 需求 | 优先级 | 验收标准 |
|---|---|---|---|
| PL-01 | 生成个人化计划前校验最低资料门槛。 | P0 | 不满足门槛时不调用个人化模板。 |
| PL-02 | 计划包含风险预算、现金边界、目标配置区间、资产类别边界、复盘规则和资料完整度。 | P0 | 每个计划均包含上述字段及生成时间。 |
| PL-03 | 所有配置权重记录模板 ID、模板版本、输入快照和规则结果。 | P0 | 相同输入可重放得到相同计划。 |
| PL-04 | 用户可查看计划形成的理由。 | P0 | 解释目标、期限、流动性、风险和约束的影响。 |
| PL-05 | 用户可保存多个版本，但只有一个当前计划。 | P0 | 新版本不覆盖历史版本，并清晰标识当前版本。 |
| PL-06 | 在没有启用产品目录时，系统先展示资产类别和选择原则。 | P0 | 页面不出现未验证的具体产品名称或代码。 |

### 5.3 持仓输入与组合诊断

支持手动输入名称或代码、金额或份额、币种和可选成本信息；CSV 导入必须允许用户在保存前确认字段映射和逐行结果。资产无法被确认时保留原始记录，标记为未解析，不能以相似资产、现金或宽基指数替代。

| ID | 需求 | 优先级 | 验收标准 |
|---|---|---|---|
| HD-01 | 用户可手动录入持仓，使用金额或份额方式。 | P0 | 保存后展示原始输入、识别状态和更新时间。 |
| HD-02 | 用户可导入 CSV，并在提交前查看字段映射和逐行错误。 | P0 | 未识别代码、缺金额和无效行逐行显示。 |
| HD-03 | 系统计算资产类别、地区、行业、币种、单一资产、集中度和重叠暴露。 | P0 | 每个诊断模块显示覆盖率和计算时间。 |
| HD-04 | 系统显示未解析权重，并在超过阈值时降低诊断可信度。 | P0 | 未解析持仓不参与低风险或正向适配结论。 |
| HD-05 | 系统将当前持仓与当前计划比较。 | P0 | 同时展示当前、目标、偏离方向与复盘原因。 |
| HD-06 | 导入或主数据解析失败返回可操作错误。 | P0 | 不允许返回空结果或静默替换资产。 |

### 5.4 研究备忘录

公开 MVP 的研究对象为已确认身份的 ETF 与公募基金。研究状态依次为：排队中、采集证据、证据不足、可审阅、风险待核验、已过期。只有可审阅状态可以进入组合影响评估。

研究备忘录固定包含：资产身份与数据时点、中性说明、已验证事实与来源、基于事实的支持论点、反方观点与冲突证据、风险与未知项、论点失效条件、研究状态与不确定性。

| ID | 需求 | 优先级 | 验收标准 |
|---|---|---|---|
| RS-01 | 用户可搜索并选择受支持资产；系统先确认资产身份。 | P0 | 无法确认身份时，不创建研究任务。 |
| RS-02 | 研究任务保存来源、发布时间、获取时间、数据时点和证据强度。 | P0 | 每项事实性主张可追溯至至少一个证据记录。 |
| RS-03 | 页面明确分开呈现事实和模型推断。 | P0 | 推断有独立标签，不能与来源事实混排。 |
| RS-04 | 备忘录包含至少一个有来源的反方观点、主要风险与失效条件。 | P0 | 缺少任一项时状态为证据不足或风险待核验。 |
| RS-05 | 研究显示证据时点与适用范围。 | P0 | 过期或覆盖不足的资料不能标记为可审阅。 |
| RS-06 | 研究页面不得输出交易指令、仓位、价格目标、收益预测或紧迫性文案。 | P0 | 自动内容检查和人工验收均不存在禁用表达。 |
| RS-07 | 用户可查看备忘录历史版本和引用快照。 | P0 | 新研究不会覆盖旧版本。 |

### 5.5 组合影响与观察清单

研究状态回答“研究资料是否足以审阅”；组合影响结论回答“它是否与用户当前计划和组合相容”。两者必须独立，研究充分不自动代表适合当前用户。

任一条件触发时，不得输出“可加入观察清单”：画像未达最低完整度；研究不是可审阅状态；资产类别超出风险预算或期限要求；纳入后突破地区、行业、币种或单一资产限额；产品可得性未验证；资产超出 P0 范围；持仓解析覆盖率不足。

| ID | 需求 | 优先级 | 验收标准 |
|---|---|---|---|
| ST-01 | 评估服务只使用版本化画像、当前计划、持仓快照和研究备忘录。 | P0 | 结果记录全部输入版本和哈希。 |
| ST-02 | 组合影响结论由确定性规则引擎生成。 | P0 | 同一输入多次运行得到同一结论和原因。 |
| ST-03 | 界面说明每条通过或阻断原因。 | P0 | 用户可看到规则名称、关联输入和安全下一步。 |
| ST-04 | 当前不匹配时，只允许保存研究、更新资料或查看通用说明。 | P0 | 不展示交易指令或绕过门槛路径。 |
| ST-05 | 观察清单记录资产、加入日期、输入版本、观察事项和过期状态。 | P0 | 用户可移除资产、关闭提醒并查看历史结论。 |
| ST-06 | 画像、持仓、计划或研究版本变化时，关联结论自动过期。 | P0 | 旧结论显示需要重新评估，不能被作为当前结果使用。 |

### 5.6 复盘、提醒与数据控制

| ID | 需求 | 优先级 | 验收标准 |
|---|---|---|---|
| RV-01 | 系统每月生成一次复盘任务，用户也可手动发起。 | P0 | 复盘展示目标、资料、持仓、偏离和研究更新。 |
| RV-02 | 在目标日期临近、画像变化、配置偏离或研究过期时建议复盘。 | P0 | 提醒不包含交易暗示。 |
| RV-03 | 用户可关闭特定或全部提醒。 | P0 | 偏好变更立即生效并保留审计记录。 |
| TR-01 | 用户可查看、修改和导出影响个人化结果的资料。 | P0 | 导出包含生成日期、输入版本和披露。 |
| TR-02 | 用户可发起删除申请。 | P0 | 产品展示删除范围、必须保留的审计类别与处理状态。 |

## 6. 业务规则、数据与系统约束

### 6.1 不可违反的业务规则

| ID | 规则 |
|---|---|
| BR-01 | 未达到最低画像门槛，不得生成标为个人化的长期计划。 |
| BR-02 | 目标配置权重只能由服务端版本化模板和规则引擎生成。 |
| BR-03 | 研究充分度与组合影响结论必须是两个独立状态。 |
| BR-04 | 组合影响评估必须冻结关键输入；任一关键输入变更即失效。 |
| BR-05 | 研究中的事实、推断、观点、风险和未知项必须明确可区分。 |
| BR-06 | 缺少来源、反方观点、风险或失效条件的研究不得进入组合影响评估。 |
| BR-07 | 数据缺失、资产身份无法确认或资料过期必须显式失败，不得伪造完整结论。 |
| BR-08 | 未启用范围不得展示不可验证的具体产品或暗示其可购买性。 |
| BR-09 | 产品不得出现交易指令、仓位建议、价格目标、收益保证和紧迫交易文案。 |
| BR-10 | 画像、计划、持仓、研究和评估结果均需记录版本与审计事件。 |
| BR-11 | 用户必须能查看与修改用于个人化结论的资料。 |
| BR-12 | 未知资产不得被填补为现金、相近资产或宽基指数。 |

### 6.2 核心数据对象

| 对象 | 关键字段 | 用途 |
|---|---|---|
| UserProfile | 地区、本位币、目标、期限、流动性、财务韧性、经验、风险、完整度、版本。 | 建档与评估输入。 |
| PortfolioPolicy | 模板 ID 与版本、配置区间、现金边界、复盘规则、生成依据、状态。 | 长期计划。 |
| HoldingSnapshot | 输入来源、持仓行、解析状态、未解析权重、估值时点、版本。 | 持仓诊断和组合影响。 |
| Instrument | 标准标识、资产类别、基金属性、币种、可用范围、数据时点。 | 主数据与产品目录。 |
| ResearchMemo | 资产、证据、事实、推断、正反观点、风险、状态、版本、过期时间。 | 研究呈现与评估前置条件。 |
| PortfolioImpactDecision | 结果、原因、规则版本、四类输入版本、输入哈希、有效期。 | 观察清单准入与审计。 |
| AuditEvent | 事件类型、主体、时间、前后版本、请求关联 ID。 | 可追溯性和问题排查。 |

### 6.3 服务边界

| 服务 | 职责 | 不负责 |
|---|---|---|
| 产品 API | 认证后的产品编排、权限与响应聚合。 | 直接计算配置权重或生成研究结论。 |
| 画像与同意服务 | 保存画像版本、计算完整度、管理同意与范围。 | 直接推荐具体资产。 |
| 计划与规则服务 | 加载模板、生成长期计划、运行组合影响规则。 | 调用 AI 覆盖确定性规则。 |
| 持仓服务 | 导入、标准化、快照和覆盖率计算。 | 静默修正无法识别的持仓。 |
| 研究编排服务 | 管理研究任务、证据、备忘录与过期状态。 | 在无证据时补全事实。 |
| 数据适配器 | 从已批准的数据源获得主数据和研究证据。 | 绕过授权或许可证限制。 |
| 审计服务 | 记录版本、输入快照和关键操作。 | 修改业务判断结果。 |

### 6.4 非功能需求

| 类别 | P0 要求 |
|---|---|
| 可解释性 | 所有计划和组合影响结论均显示输入版本、规则原因和生成时间。 |
| 数据质量 | 每项资产或市场数据均显示来源、市场、币种和数据时点；无法确认时返回失败。 |
| 审计性 | 计划、研究和评估结果保留版本、输入快照和审计事件。 |
| 安全与隐私 | 最小化采集、加密存储、权限控制、访问审计；不保存券商密码。 |
| 性能 | 建档与本地规则计算提供即时反馈；研究任务以异步状态和进度反馈呈现。 |
| 无障碍与本地化 | 首发中文，关键文本可本地化；金额、日期和币种遵循用户区域格式。 |
| 失败可见性 | 导入、数据、研究和评估失败必须返回具体原因、受影响对象和下一步。 |

## 8. 验收测试与上线门槛

### 8.1 核心验收场景

| 场景 | 前置条件 | 操作 | 预期结果 |
|---|---|---|---|
| A1 建档成功 | 已选择启用范围。 | 完成所有关键问题。 | 生成可保存计划，显示模板、配置区间、理由与版本。 |
| A2 建档信息不足 | 缺少目标日期或流动性信息。 | 尝试生成计划。 | 显示缺项与通用示例，不显示个人化结论。 |
| A3 持仓导入含错误 | CSV 含未知代码和无金额行。 | 上传并预览。 | 每行显示状态；未知项不被替换。 |
| A4 持仓覆盖不足 | 未解析权重大于配置阈值。 | 查看诊断或评估。 | 标记覆盖不足；评估要求补充信息。 |
| A5 研究充分 | 已确认 ETF，证据完整且未过期。 | 发起研究。 | 备忘录包含来源、事实、推断、反方、风险和失效条件。 |
| A6 研究不足 | 缺少反方观点或关键来源。 | 发起研究。 | 状态为证据不足或风险待核验，不可进入评估。 |
| A7 评估通过 | 画像完整，资产满足风险预算与可得性。 | 查看组合影响。 | 可加入观察清单，展示原因与输入版本。 |
| A8 评估阻断 | 资产将突破地区或资产类别限额。 | 查看组合影响。 | 当前不匹配，不显示交易指令。 |
| A9 结果过期 | 用户修改期限、风险资料或持仓。 | 返回观察清单。 | 关联结论标为需要重新评估。 |
| A10 未启用范围 | 用户选择没有产品映射的范围。 | 查看研究入口。 | 仅显示通用框架，不显示未验证产品。 |

### 8.2 上线阻断项

- 任何路径绕过最低画像门槛并产生个人化计划；
- 任何备忘录缺少反方、风险、失效条件或证据仍可进入组合影响评估；
- 任意正向评估结果未记录输入版本、规则原因与生成时间；
- 未解析持仓被静默归类或在诊断中隐藏；
- 产品出现交易指令、仓位、价格目标、收益保证或时间紧迫性文案；
- 未启用范围展示未验证的具体产品；
- 用户不能查看、修改或导出影响其个人化结论的资料。

## 9. 依赖、风险与待决策项

### 9.1 外部依赖

| 依赖 | 影响 | 所有者 |
|---|---|---|
| 证券与基金主数据 | 资产身份确认、持仓分类和风险分析。 | 数据团队 |
| 市场和基金静态数据许可 | 数据时点、研究质量和对外展示范围。 | 数据与法务 |
| 产品目录与范围配置 | 具体产品展示和访问控制。 | 产品与合规 |
| 引用存档能力 | 研究可验证性与内容审查。 | 数据与平台 |
| LLM 与受控检索服务 | 研究整理、事实与推断分离、反方组织。 | AI 与平台 |
| 版本与审计存储 | 计划、研究和评估的重放与追溯。 | 平台团队 |

### 9.2 主要风险与缓解措施

| 风险 | 缓解措施 |
|---|---|
| 将研究包装成隐性荐股 | 固定文案规则，分离研究与评估，设置规则门槛并进行合规审核。 |
| AI 编造或遗漏证据 | 使用受控数据适配器、来源快照、证据门槛和显式失败状态。 |
| 产品可得性误判 | 按范围配置目录、灰度启用；未启用时仅显示通用资产类别。 |
| MVP 复杂度失控 | 默认基金与 ETF，推迟个股、账户聚合和交易连接。 |
| 用户过度依赖结果 | 强制展示不确定性、反方观点和复盘规则；不执行交易。 |
| 数据和开源许可证不匹配 | 数据源与组件均需在接入前登记授权边界。 |

### 9.3 上线前待决策项

1. 首个启用的市场、用户地区和基金或 ETF 产品目录；
2. 产品的法律定位：研究教育工具，或在持牌合作下提供其他服务；
3. 主数据、市场数据、基金数据和研究资料的供应商及许可范围；
4. 未解析持仓权重、研究过期与配置偏离的具体阈值；
5. 观察清单、复盘提醒和数据保留的默认策略；
6. 是否在封闭 Beta 纳入白名单个股及对应的额外审核要求。

## 10. 术语

| 术语 | 定义 |
|---|---|
| 长期计划 | 包含目标、期限、风险预算、目标配置、边界和复盘规则的可保存计划。 |
| 风险承受能力 | 由收入、负债、应急资金和期限等客观因素决定的损失承受能力。 |
| 风险偏好 | 用户主观上愿意承受波动的程度。 |
| 目标配置 | 为长期目标设定的资产类别配置区间，不是具体证券清单。 |
| 研究状态 | 表示研究资料质量和时效性的状态，不表示是否适合用户。 |
| 组合影响结论 | 在用户画像、长期计划、持仓和范围约束下，对候选资产进行的规则化评估。 |
| 观察清单 | 通过资料和组合影响门槛、可继续跟踪的资产集合，不是交易清单。 |
| 未解析持仓 | 系统无法可靠确认身份或分类的原始持仓记录。 |
