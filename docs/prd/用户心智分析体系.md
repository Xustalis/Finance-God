## 三、理解层：心智建模

### 3.1 风险画像模块（Trait层 — 相对稳定）

**评估工具组合**：

| 量表/方法                    | 维度                     | 题目数   | 更新频率 |
| :--------------------------- | :----------------------- | :------- | :------- |
| **Grable & Lytton (GL-RTS)** | 财务风险承受力           | 13题     | 季度复评 |
| **GDMS决策风格量表**         | 理智/直觉/依赖/逃避/冲动 | 25题     | 半年     |
| **Financial Anxiety Scale**  | 财务焦虑程度             | 5-8题    | 月度     |
| **行为揭示偏好**             | 实际风险行为 vs 自述偏好 | 持续计算 | 实时     |

**关键创新 — "言行一致性"校正**：

```python
def calibrated_risk_tolerance(user):
    stated = questionnaire_score(user)        # 问卷自述 (0-100)
    revealed = behavior_implied_risk(user)    # 行为揭示 (0-100)
    
    # 行为揭示基于：实际持仓波动率、止损点设置、
    # 市场下跌时的操作（减仓=低容忍、加仓=高容忍）
    
    consistency = 1 - abs(stated - revealed) / 100
    
    # 一致性低时偏向行为数据（更真实）
    weight_behavior = 0.3 + 0.4 * (1 - consistency)
    weight_stated = 1 - weight_behavior
    
    calibrated = weight_stated * stated + weight_behavior * revealed
    return calibrated, consistency
```

**研究依据**：剑桥大学2022年报告显示35%用户在问卷中不诚实，行为数据更能反映真实承受力。

### 3.2 情绪识别模块（State层 — 动态变化）

**多源信号融合模型**：

```text
输入信号：├── 文本情绪 (FinBERT2) → [fear, anxiety, greed, confidence, calm] 向量├── 行为特征 →│   ├── 登录频率异常度 (z-score vs 个人基线)│   ├── 持仓查看强迫度 (单日查看次数 / 历史均值)│   ├── 交易冲动指数 (撤单率 + 深夜交易比例)│   └── 信息焦虑度 (负面新闻搜索频率)└── 市场环境 → 市场恐慌/贪婪指数 (0-100)
融合方式：多层感知机 or 注意力机制加权融合 → 输出综合情绪状态向量
```

**情绪状态定义（基于心理学情绪周期模型）**：

| 状态                  | 情绪分数区间 | 行为特征                     | 典型市场环境 |
| :-------------------- | :----------- | :--------------------------- | :----------- |
| **平静** (Calm)       | 60-80        | 正常频率操作、理性决策       | 震荡市       |
| **乐观** (Optimistic) | 80-90        | 适度加仓、关注机会           | 温和上涨     |
| **贪婪** (Greedy)     | 90-100       | 追涨、加杠杆、忽视风险       | 快速上涨     |
| **焦虑** (Anxious)    | 40-60        | 频繁查看、犹豫不决           | 初步下跌     |
| **恐惧** (Fear)       | 20-40        | 想割肉、深夜查看、搜索"止损" | 持续下跌     |
| **恐慌** (Panic)      | 0-20         | 不计成本卖出、频繁登录       | 暴跌/黑天鹅  |

### 3.3 认知偏差检测模块

**五大核心偏差的量化检测规则**：

#### 损失厌恶 (Loss Aversion)

```python
def loss_aversion_score(user):
    """基于Prospect Theory: λ参数估计"""
    # 方法1: 交易行为推断
    gains = positions_with_gain(user)
    losses = positions_with_loss(user)
    
    # 盈利头寸平均持有天数 vs 亏损头寸平均持有天数
    avg_hold_gain = mean(holding_days(gains))
    avg_hold_loss = mean(holding_days(losses))
    
    # 损失厌恶者倾向于：快速卖掉盈利，长期持有亏损
    ratio = avg_hold_loss / (avg_hold_gain + 1)
    
    # 方法2: 问卷校准 (确定性等价法)
    # "50%概率赢200 vs 确定赢X，你选哪个？" → 拟合λ
    
    # 综合评分
    score = sigmoid((ratio - population_median) / population_std)
    return score  # >0.7 = 高损失厌恶
```

**触发条件**：市场连续下跌 + 用户持有浮亏头寸
**干预策略**：推送长期回报数据、历史亏损恢复案例、建议设置自动止损

#### 过度自信 (Overconfidence)

```python
def overconfidence_score(user):
    """高频交易 + 高杠杆 + 实际收益不佳 = 过度自信"""
    trade_freq = monthly_trades(user)
    leverage_usage = avg_leverage(user)
    actual_alpha = portfolio_alpha(user, benchmark="CSI300")
    
    # 过度自信特征：交易频繁但未产生超额收益
    freq_z = zscore(trade_freq, population_distribution)
    lev_z = zscore(leverage_usage, population_distribution)
    
    # 自信度与实际表现的背离
    raw_score = 0.4 * freq_z + 0.3 * lev_z - 0.3 * max(actual_alpha, 0)
    return sigmoid(raw_score)
```

**触发条件**：牛市环境 + 连续盈利后
**干预策略**：展示历史回撤数据、提示"运气vs技能"、建议分散化

#### 从众效应 (Herding)

```python
def herding_score(user, market_data):
    """用户交易方向与群体资金流高度一致 + 缺乏独立分析"""
    user_buys = user.buy_events(last_30_days)
    
    # 计算用户买入时点与市场热门资金流入峰值的时间相关性
    hot_sector_flow = market_data.sector_net_inflow()
    temporal_corr = correlation(user_buys.timestamps, hot_sector_flow.peaks)
    
    # 热门资产集中度
    hot_stock_ratio = user.portfolio_weight_in_trending_stocks()
    
    # 缺乏基本面研究行为
    research_behavior = user.research_report_reads(last_30_days)
    
    score = 0.4 * temporal_corr + 0.4 * hot_stock_ratio - 0.2 * min(research_behavior / 5, 1)
    return sigmoid(score)
```

#### 处置效应 (Disposition Effect)

```python
def disposition_effect_score(user):
    """Odean指标: PGR - PLR"""
    # PGR = 实现盈利次数 / 账面盈利机会总数
    PGR = realized_gain_count / (paper_gain_opportunities + 1e-6)
    # PLR = 实现亏损次数 / 账面亏损机会总数  
    PLR = realized_loss_count / (paper_loss_opportunities + 1e-6)
    
    disposition = PGR - PLR  # >0 表示处置效应存在
    z_score = (disposition - population_mean) / population_std
    return sigmoid(z_score)
```

#### 锚定效应 (Anchoring)

```python
def anchoring_score(user):
    """对买入价/历史高点过度依赖，对新信息反应迟缓"""
    scores = []
    for pos in user.positions:
        anchor_price = pos.entry_price  # 锚定点：买入价
        
        # 当出现重大基本面变化后，用户调仓的反应时间
        fundamental_events = significant_events(pos.symbol)
        for event in fundamental_events:
            reaction_time = time_to_first_action(user, pos, after=event.time)
            price_deviation = abs(current_price - anchor_price) / anchor_price
            
            # 价格已大幅偏离锚点但仍不调整 → 高锚定
            if price_deviation > 0.2 and reaction_time > 7:  # 7天无反应
                scores.append(1.0)
            else:
                scores.append(reaction_time / 14)  # 归一化
    
    return mean(scores) if scores else 0.0
```

### 3.4 心理状态识别（HMM模型）

**隐马尔可夫模型设计**：

```text
隐状态 (Hidden States):  S1: Calm (平静理性)  S2: Concerned (轻度担忧)  S3: Anxious (焦虑)  S4: Panic (恐慌)  S5: Euphoric (狂热)
```

## 四、预测层：风险预警

### 4.1 行为序列预测模型

**架构**：Transformer Encoder + 时间注意力

```text
输入：近30天行为事件序列 (event_type, timestamp, context)     + 当前心智状态向量     + 市场环境特征
输出：  - 未来7天情绪化交易概率 (0-1)  - 状态转移预测 (Calm→Panic 概率)  - 冲动操作风险等级 (low/medium/high/critical)
```

### 4.2 异常检测

**方法**：LSTM-AutoEncoder

- 输入：用户行为时间序列（登录频率、交易频率、查看频率等）
- 正常行为重构误差低，异常行为重构误差高
- 当重构误差 > 阈值 → 触发"心理波动"预警

## 五、干预层：策略与时机

### 5.1 干预优先级矩阵

| 紧急度      | 触发条件                         | 干预方式                              | 响应时间 |
| :---------- | :------------------------------- | :------------------------------------ | :------- |
| **P0 紧急** | panic_score > 0.9 + 大额卖出操作 | 交易冷静期(强制等待15分钟) + 安抚对话 | 即时     |
| **P1 高**   | anxiety > 0.7 + 频繁登录         | 推送安抚内容 + 历史案例 + 策略微调    | 5分钟内  |
| **P2 中**   | 偏差检测触发 (如herding > 0.7)   | 教育提示 + 风险数据展示               | 当日     |
| **P3 低**   | 轻度焦虑 / 日常维护              | 定期心理报告 + 正向反馈               | 次日     |

### 5.2 干预策略库（基于CBT + 行为金融学）

| 偏差/状态        | 干预技术     | 具体实现                                      |
| :--------------- | :----------- | :-------------------------------------------- |
| 恐慌卖出倾向     | CBT认知重构  | "让我们看看历史上类似跌幅后的恢复数据..."     |
| 过度自信         | 概率教育     | "你过去6个月的预测准确率是42%，而你感觉是80%" |
| 从众追热         | 反向数据     | "当前XX板块散户买入占比已达历史95%分位..."    |
| 损失厌恶(割肉难) | 沉没成本教育 | "不管买入价多少，关键是这个价格值不值得持有"  |
| 决策疲劳         | System 2唤起 | 强制确认步骤 + 冷静倒计时 + "今日已交易5次"   |

### 5.3 Dual Process Theory 应用

```text
System 1 场景（允许快速决策）：  - 自动定投执行  - 已设定规则的止损触发  - 小额操作
System 2 场景（强制慢思考）：  - 单笔 > 总资产10%的交易 → 额外确认 + 展示影响分析  - 加杠杆操作 → 显示历史爆仓案例 + 倒计时确认  - 市场极端时的大额操作 → "你确定这不是情绪驱动？" + 24h延迟选项
```
