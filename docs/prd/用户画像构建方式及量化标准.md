# User Intelligence Engine（用户智能画像系统）

## 一、系统架构

| Layer                       | 数据来源    | 获取方式              | 数据特点      | 更新频率 | 可信度   |
| --------------------------- | ------- | ----------------- | --------- | ---- | ----- |
| **Static Profile（静态画像）**    | 用户填写    | Selection（选择题）    | 客观事实      | 很低   | ⭐⭐⭐⭐  |
| **Cognitive Profile（心智画像）** | AI 语音聊天 | AI Conversation   | 心理、目标、价值观 | 中    | ⭐⭐⭐   |
| **Behavior Profile（行为画像）**  | 用户真实行为  | Behavior Tracking | 最真实       | 持续更新 | ⭐⭐⭐⭐⭐ |
| **Confidence Layer（可信度层）**  | 系统计算    | Data Fusion       | 判断画像可靠程度  | 实时   | ——    |

---

## 二、Static Profile（静态画像）

> **回答：“这个用户是谁？”（Who）**

| 一级维度 | 二级指标    | 数据类型         | 量化方式                                 |
| ---- | ------- | ------------ | ------------------------------------ |
| 基础信息 | 年龄      | Number       | 实际年龄                                 |
| 基础信息 | 性别      | Enum         | Male / Female / Other                |
| 基础信息 | 地区      | Enum         | 城市等级                                 |
| 基础信息 | 职业      | Enum         | Student / Employee / Self-employed 等 |
| 基础信息 | 学历      | Enum         | High School ~ PhD                    |
| 财富情况 | 年收入     | Number       | RMB                                  |
| 财富情况 | 月现金流    | Number       | 收入 - 支出                              |
| 财富情况 | 总资产     | Number       | RMB                                  |
| 财富情况 | 流动资产    | Number       | RMB                                  |
| 财富情况 | 负债      | Number       | RMB                                  |
| 财富情况 | 每月可投资金额 | Number       | RMB                                  |
| 投资经历 | 投资年限    | Enum         | 0、1~3、3~5、5+                         |
| 投资经历 | 投资产品    | Multi Select | 股票、ETF、基金等                           |
| 投资经历 | 金融知识    | Self Rating  | 1~5                                  |
| 投资目标 | 第一目标    | Enum         | 财富自由、养老等                             |
| 投资目标 | 投资期限    | Enum         | 小于 1 年、1~3 年、3~10 年、10 年以上           |

### 系统自动计算

| 指标                  | 公式或计算方式        |
| ------------------- | -------------- |
| Wealth Level        | 根据资产规模进行分层     |
| Debt Ratio          | `Debt / Asset` |
| Cash Flow Stability | 根据月现金流波动率计算    |
| Experience Score    | 投资年限 + 产品复杂度   |

## 三、Cognitive Profile（AI 心智画像）

> **回答：“用户为什么这样投资？”（Why）**

AI 通过自然聊天持续推断用户的目标、动机、情绪、风险认知与思维方式。

它不是一次性问卷，而是会随着用户对话不断更新。

---

### 1. Goal（目标）

> 参考：Goal Setting Theory

| 指标                   | AI 判断依据        | Score |
| -------------------- | -------------- | ----- |
| Financial Freedom    | 高频提及自由、退休、摆脱工作 | 0~100 |
| Wealth Growth        | 明显追求收益最大化      | 0~100 |
| Capital Preservation | 强烈关注保本与避免亏损    | 0~100 |
| Education Planning   | 关注教育支出和子女教育    | 0~100 |
| Retirement Planning  | 关注养老金和退休规划     | 0~100 |

---

### 2. Motivation（动机）

> 参考：Self-Determination Theory（Deci & Ryan）

| 动机              | AI 判断依据         |
| --------------- | --------------- |
| Freedom（自由）     | 希望摆脱工作、拥有时间自由   |
| Security（安全）    | 担心未来不确定性，希望获得保障 |
| Achievement（成就） | 喜欢战胜市场、证明能力     |
| Status（身份）      | 将财富视为成功和身份象征    |
| Family（家庭）      | 为家人、子女或家庭未来投资   |

量化示例：

```text
Freedom Score: 82
Security Score: 55
Achievement Score: 38
```

---

### 3. Emotion（情绪）

> 参考：Behavioral Finance + Prospect Theory

| 指标         | AI 分析内容 | Score |
| ---------- | ------- | ----- |
| Anxiety    | 焦虑程度    | 0~100 |
| Fear       | 恐惧程度    | 0~100 |
| Confidence | 自信程度    | 0~100 |
| Optimism   | 乐观程度    | 0~100 |
| Stress     | 压力程度    | 0~100 |

示例：

> 用户最近频繁提及亏损、担心和后悔。

系统可能判断：

```text
Fear Score ↑
Anxiety Score ↑
Confidence Score ↓
```

---

### 4. Risk Cognition（风险认知）

> 参考：Prospect Theory（前景理论）

AI 不直接询问：

> “你的风险承受能力高吗？”

而是通过情景问题进行推断，例如：

* 如果一个月亏损 20%，你会怎么做？
* 如果市场连续下跌一年，你会怎么做？
* 如果当前收益达到 30%，你会卖出还是继续持有？
* 如果亏损扩大，你会止损、补仓还是观望？

系统计算：

| 指标                | 含义           | Score |
| ----------------- | ------------ | ----- |
| Risk Tolerance    | 风险承受能力       | 0~100 |
| Loss Aversion     | 损失厌恶程度       | 0~100 |
| Recovery Ability  | 面对回撤后的恢复能力   | 0~100 |
| Panic Probability | 市场下跌时恐慌操作的概率 | 0~100 |

---

### 5. Thinking Style（投资认知）

> 参考：Dual Process Theory（System 1 / System 2）

| 指标                 | 含义               | Score |
| ------------------ | ---------------- | ----- |
| Data Driven        | 是否依赖数据决策         | 0~100 |
| News Driven        | 是否容易受新闻影响        | 0~100 |
| KOL Driven         | 是否容易受财经博主或意见领袖影响 | 0~100 |
| AI Trust           | 对 AI 建议的信任程度     | 0~100 |
| Emotional Decision | 情绪化决策倾向          | 0~100 |
| Rational Decision  | 理性分析倾向           | 0~100 |

示例：

```text
AI Trust: 73
Data Driven: 88
Emotional Decision: 26
```

## 四、Behavior Profile（行为画像）

> **回答：“用户真正是怎么做的？”（How）**

行为画像的可信度最高，因为实际行为通常比用户口头表达更接近真实偏好。

---

### A. Historical Behavior（历史行为）

> 参考：Behavioral Finance

| 指标                  | 数据来源   | 单位 |
| ------------------- | ------ | -- |
| Holding Period      | 平均持仓时间 | 天  |
| Trading Frequency   | 年交易次数  | 次  |
| Win Rate            | 盈利交易占比 | %  |
| Max Drawdown        | 最大回撤   | %  |
| Position Adjustment | 调仓频率   | 次  |
| Stop Loss           | 止损次数   | 次  |
| Average Holding     | 平均仓位   | %  |

系统自动生成：

| 指标                    | Score |
| --------------------- | ----- |
| Patience Score        | 0~100 |
| Discipline Score      | 0~100 |
| Volatility Preference | 0~100 |
| Trading Style         | 分类    |

Trading Style 可分为：

* Long-term Investor
* Swing Trader
* Short-term Trader
* Trend Follower
* Value Investor
* High-frequency Trader

---

### B. Product Behavior（产品行为）

系统持续观察用户在产品中的真实行为。

| 行为                    | 数据             |
| --------------------- | -------------- |
| Daily Open Count      | 每天打开 App 的次数   |
| Portfolio Check       | 每天查看收益的次数      |
| News Reading Time     | 阅读财经新闻的时长      |
| Strategy Modification | 修改投资策略的次数      |
| Cancel AI Advice      | 拒绝或取消 AI 建议的次数 |
| View Loss             | 查看亏损页面的频率      |
| View Hot Stocks       | 查看热门股票的频率      |

系统自动生成：

| 指标              | Score |
| --------------- | ----- |
| AI Trust Score  | 0~100 |
| Long-term Score | 0~100 |
| Impulsiveness   | 0~100 |
| Execution Score | 0~100 |
| FOMO Score      | 0~100 |
| Loss Obsession  | 0~100 |

## 五、Confidence Layer（可信度层）

同一个用户特征，可能来自不同数据来源。

不同来源的可信度不同，因此系统不能只保存最终结果，还需要保存：

```text
Value
Confidence
Source
Last Updated
```

### 来源权重

| 来源              | 权重  |
| --------------- | --- |
| Behavior        | 50% |
| AI Conversation | 35% |
| Selection       | 15% |

最终分数可采用加权计算：

```text
Final Score
= Behavior Score × 50%
+ AI Conversation Score × 35%
+ Selection Score × 15%
```

### 示例

| 用户特征   | Selection | AI Conversation | Behavior | Final |
| ------ | --------: | --------------: | -------: | ----: |
| 风险承受能力 |        90 |              70 |       25 |    42 |

计算方式：

```text
Final Risk Score
= 90 × 15%
+ 70 × 35%
+ 25 × 50%

= 13.5 + 24.5 + 12.5
= 50.5
```

因此，按照上述权重，示例中的最终结果应为：

```text
Risk Score = 51
```

这说明：

> 用户主观上认为自己风险承受能力很高，但对话和真实行为显示，其实际风险承受能力较低。

## 八、核心逻辑

```text
Selection
    ↓
获取客观信息
    ↓
AI Conversation
    ↓
理解目标、动机、情绪与风险认知
    ↓
Behavior Tracking
    ↓
使用真实行为验证用户表达
    ↓
Confidence Layer
    ↓
融合不同来源并计算可信度
    ↓
Unified User Profile
    ↓
生成个性化投资策略
    ↓
持续观察和更新画像
```

---

## 九、核心原则

1. **用户画像不是一次性问卷，而是持续更新的动态模型。**
2. **用户说自己会怎么做，不一定等于真正会怎么做。**
3. **行为数据的权重通常高于自我描述。**
4. **敏感心理特征不能只靠一次对话判断。**
5. **所有画像结论都应记录来源、时间和可信度。**
6. **用户可以查看、纠正和删除自己的画像信息。**
7. **涉及投资建议时，画像只能辅助决策，不能替代适当性审查和风险控制。**
