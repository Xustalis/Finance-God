# T10 过程与证据页面

## 页面定位

当前页面定位为 **AI 研究、数据来源、规则命中、版本关系和执行事件的统一追溯界面**。

普通用户从各页面打开证据抽屉，高级用户可进入完整过程页，内部人员在权限允许时查看诊断信息。三种模式使用同一底层事实，但披露深度不同。

## 与 AI 辅助交易台的关系

- AI 辅助交易台中的“分析依据”“查看过程”和错误详情均打开本页或抽屉。
- 页面只解释已有产物和事件，不生成新的交易建议或订单。
- 用户可从证据返回原页面和原对象版本。

## 页面原型

```text
普通模式：原任务页面 + 480 px 只读证据抽屉
┌───────────────────────────────────────────┬──────────────────────────┐
│ 原页面和当前对象版本保持可见              │ 结论版本                 │
│                                           │ 事实 / 推断              │
│                                           │ 反方 / 未知              │
│                                           │ 来源时点 / 规则影响      │
└───────────────────────────────────────────┴──────────────────────────┘

高级模式：3/9 独立页面
┌───────────────────┬──────────────────────────────────────────────────┐
│ 对象 / 版本 / 节点│ 单一证据正文：来源、过程、规则、事件、错误      │
└───────────────────┴──────────────────────────────────────────────────┘
```

布局约束：

- 普通用户默认使用 480 px 右侧只读抽屉，保持原任务和版本可见；
- 抽屉固定顺序为结论版本 → 事实/推断 → 反方/未知 → 来源时点 → 规则影响；
- 禁止抽屉嵌套；需要版本比较、跨对象检索或导出时进入高级独立页；
- 高级独立页使用 3/9，左侧只负责对象/版本/节点导航，右侧同一时间只展示一种详情；
- 内部错误栈和敏感诊断不得进入普通抽屉。

## 页面功能

| 页面区域 | 功能模块 | 页面功能 | 主要呈现信息 | 用户操作 | AI处理 |
| --- | --- | --- | --- | --- | --- |
| 顶部栏 | 对象定位 | 指定当前追溯对象 | 对象类型、ID、版本、来源页面 | 复制链接、返回 | 读取对象关系 |
| 顶部栏 | 展示模式 | 切换普通和高级信息 | 普通、高级；内部模式由权限决定 | 切换 | 按模式生成摘要 |
| 顶部栏 | 完整度 | 展示证据是否完整 | 缺失、冲突、过期、已撤回来源 | 查看问题 | 标记结论限制 |
| 结论区 | 结论摘要 | 展示被解释的原结论 | 结论、适用周期、置信度、生成时间 | 查看原对象 | 不重新生成结论 |
| 结论区 | 事实 | 列出可验证输入 | 数值、文本、来源、时点、适用范围 | 展开来源 | 摘要事实 |
| 结论区 | 推断 | 展示由事实得出的判断 | 推断链、支持事实、反方事实 | 展开链路 | 解释推理 |
| 结论区 | 假设 | 展示模型和情景假设 | 费用、滑点、周期、市场条件 | 查看影响 | 解释假设敏感性 |
| 结论区 | 未知项 | 展示无法验证的信息 | 缺失字段、冲突、影响范围 | 反馈、请求更新 | 不填补未知事实 |
| 来源区 | 来源列表 | 查看所有输入来源 | 名称、类型、时间、版本、许可、状态 | 筛选、打开 | 生成来源摘要 |
| 来源区 | 数据血缘 | 查看数据如何进入结论 | 原始数据、标准化、派生指标、产物 | 展开节点 | 解释转换 |
| 来源区 | 冲突对比 | 并列不同来源的值 | 来源 A/B、时间、差异、采用规则 | 请求复核 | 总结冲突，不选权威 |
| 来源区 | 撤回与过期 | 标记不可继续使用的来源 | 撤回原因、过期时间、受影响对象 | 查看影响 | 使相关结论过期 |
| Agent 区 | 工作流图 | 展示 Agent 任务关系 | Planner、Data、Research、Factor、Strategy、Backtest、Report | 展开节点 | 生成任务摘要 |
| Agent 区 | 任务状态 | 查看每步输入输出 | 排队、运行、完成、失败、阻断、重试 | 查看详情 | 不隐藏失败 |
| Agent 区 | 反馈轮次 | 展示产物如何修订 | 反馈来源、修改字段、版本 | 比较 | 总结差异 |
| Agent 区 | 错误详情 | 展示失败信息 | 错误类型、时间、Trace ID、受影响范围 | 重试允许的任务、复制 ID | 解释错误，不伪造恢复 |
| 规则区 | 授权命中 | 展示使用的授权字段 | 版本、字段、限制、结果 | 进入 T00 | 解释规则 |
| 规则区 | 风控结果 | 展示通过、警告和阻断 | 规则码、输入、阈值、结果、版本 | 查看明细 | 转为通俗说明 |
| 事件区 | 用户确认 | 展示用户明确操作 | 页面、对象、时间、确认内容 | 查看 | 不生成确认记录 |
| 事件区 | 执行事件 | 展示订单事实时间线 | 提交、受理、成交、撤单、拒单 | 进入 T07 | 总结，不修改 |
| 版本区 | 版本比较 | 比较任意两个版本 | 新增、删除、修改字段、原因 | 选择版本 | 生成差异摘要 |
| 导出区 | 证据包 | 导出可审计材料 | 对象、来源、规则、事件、哈希/版本 | 选择范围、导出 | 生成目录和说明 |

## 展示模式

| 模式 | 面向对象 | 默认内容 | 隐藏内容 |
| --- | --- | --- | --- |
| 普通模式 | 普通投资用户 | 来源、时点、事实/推断、反方、未知、规则原因 | 内部提示词、敏感配置、错误栈 |
| 高级模式 | 专业用户、研发协作 | Agent 节点、产物版本、反馈轮次、规则明细 | 密钥、其他用户数据 |
| 内部模式 | 经授权的运营、风控、技术 | Trace、错误栈、服务版本、人工处置 | 超出岗位权限的数据 |

## 关键状态

| 状态 | 页面表现 | 用户可执行操作 |
| --- | --- | --- |
| 证据完整 | 展示完整结论链 | 查看、导出 |
| 证据缺失 | 标出缺项和受影响结论 | 请求更新 |
| 来源冲突 | 并列冲突，不隐藏 | 请求复核 |
| 结论过期 | 保留历史并禁用继续生成草稿 | 返回更新 |
| 任务失败 | 显式错误和 Trace ID | 重试允许的任务 |
| 权限不足 | 说明当前可见范围 | 返回，不泄露字段名和值 |
| 对象不存在 | 显式错误，不跳到类似对象 | 返回来源页面 |

## 开源项目补充

参考 TradingAgents、FinRobot 和 OpenBB，证据页增加 Agent 拓扑、检查点/恢复、Provider 元数据和“原始事实—清洗字段—确定性计算—LLM 解释”数值血缘。重新运行必须创建新 Run 或明确恢复点，不能伪装成原运行连续成功。详细依据见 [开源项目参考与页面补充](02_开源项目参考与页面补充.md#59-t10-过程与证据)。

## AI 与证据边界

| AI可以做 | AI不能做 |
| --- | --- |
| 总结来源、推断链和版本差异 | 伪造缺失来源 |
| 将规则与错误转换为通俗说明 | 隐藏失败、冲突和未知项 |
| 帮助检索对象和事件 | 修改授权、规则或执行事实 |
| 解释假设和适用范围 | 将模型推断标记为事实 |
| 生成证据包目录 | 暴露内部密钥、提示词或其他用户数据 |

## 导航关系

- 入口：T01 风险/研究卡、T03 分析依据、T04 计划证据、T07 校验和事件、T08 数据来源、T09 复盘证据。
- 出口：返回发起页面的同一对象和版本；授权进入 T00；订单进入 T07。
- 深链接必须包含对象类型、ID 和版本，不能只靠页面内临时状态。

## 验收标准

- 任一 AI 结论都能区分事实、推断、假设和未知项。
- 来源过期或撤回后，所有受影响对象可被列出。
- Agent 失败和规则阻断不会被通用“处理中”掩盖。
- 普通模式不会泄露敏感内部实现，高级模式也不能查看其他用户数据。
- 版本比较使用不可变历史，不根据当前值重建。
- 导出的证据包包含对象版本和生成时间。
