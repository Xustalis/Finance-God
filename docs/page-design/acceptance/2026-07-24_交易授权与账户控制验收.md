# 交易授权与账户控制验收

> 范围：T00「交易授权与账户控制」。后端新增持久化 `investment_mandate`（版本化，插入新版本不覆盖历史）、`MandateService`、`PersistentAuthorizationProvider` 与默认 L0 active mandate 自动创建；`SimulationRiskAdapter` 接入授权做**真实拦截**（状态/side/order_type/市场/单笔上限任一不满足即风控 BLOCKED + HARD reason）；新增 `/mandate/*` API。前端在 `/settings`「我的」页新增「交易授权」分区（不新建路由），提供状态摘要、编辑器、影响面、版本历史与暂停/恢复/撤销。授权为**仿真业务数据**，页面不存储任何经纪商凭据。
>
> 结果只能填写：`通过`、`失败`、`不适用`。

## 10. 最终结论

- 验收结论：**通过（工程与专项验收）**。后端 22 项新增单测 + 171 项回归全绿，前端 81 项单测 + type-check + lint 全绿；T00 五项重点专项均通过。
- 未通过项目：`1440/1024/<1024 px 浏览器测试`标记为失败——本轮未执行浏览器实测。
- 已知限制：
  - 桌面宽度与视觉密度未做浏览器实测，仅复用 SettingsView 既有布局与令牌，未新增布局代码。

  - Alembic 迁移在真实本地 postgres 上 upgrade/downgrade 均验证成功（0009↔0010↔0011 链路完整、当前 head=0011）。
