# 共享 AI 侧栏验收

> 验收范围：认证后的全部桌面路由共享同一个右侧 AI sidebar/rail。结果只能填写 `通过`、`失败` 或 `不适用`。

## 1. 任务与信息架构

| 验收项 | 结果 | 证据 |
| --- | --- | --- |
| 页面首屏能识别用户任务、当前对象和主要状态 | 通过 | 1440 浏览器快照显示“当前标的 · 000001.SZ / symbol”和 AI 空闲态。 |
| 每个区域只有一个明确职责 | 通过 | `App.vue` 只负责共享壳与路由上下文；`AiSidebar.vue` 负责 AI 状态与交互。 |
| 主操作唯一且结果可预测 | 通过 | 侧栏主操作为“分析当前对象”，折叠按钮只改变工作区布局。 |
| 页面与上一步、下一步对象保持连续 | 通过 | `ai-sidebar.spec.ts` 验证追问草稿在收起及上下文切换后保留。 |
| 没有无功能价值的区域或卡片 | 通过 | 侧栏仅包含对象、运行状态、结果证据和追问输入。 |

## 2. 文案

| 验收项 | 结果 | 证据 |
| --- | --- | --- |
| 标题直接说明区域内容或操作 | 通过 | “AI 侧栏”“当前对象”“支持证据（事实）”均直接命名功能。 |
| 辅助文字说明范围、操作、时效或决策价值 | 通过 | 空闲态说明结果来自 Multi-Agent 且不在浏览器派生。 |
| 删除了口号、隐喻、设计说明和重复文案 | 通过 | 侧栏没有营销口号或设计自述。 |
| 按钮使用“动词 + 对象/结果” | 通过 | “分析当前对象”“重新分析当前对象”“展开/收起 AI 侧栏”。 |
| 错误、空状态和恢复条件表达明确 | 通过 | 未选对象、运行中、错误码、无结果均有显式状态。 |

## 3. 真实数据

| 验收项 | 结果 | 证据 |
| --- | --- | --- |
| 市场数据来自 PandaData，不含硬编码行情 | 不适用 | 本改动不创建或修改市场数据。 |
| 显示来源、供应商时间、实际频率和新鲜度 | 不适用 | 属于各页面行情区职责；本改动只同步对象标识。 |

## 10. 最终结论

- 验收结论：共享 AI 侧栏功能范围通过；既有交易台控制台资源错误仍需单独收口。
- 未通过项目：交易台既有控制台资源错误。
- 已知限制：追问文本作为下一次真实 AI 运行的补充上下文，不提供独立发送动作；AI 失败时不生成替代结论。
- 复现命令或测试入口：
  - `cd frontend && npm run test -- --run src/tests/ai-sidebar.spec.ts src/tests/app-ai-shell.spec.ts`
  - `output/playwright/ai-sidebar-desk-1440.png`
  - `output/playwright/ai-sidebar-desk-1024.png`
