# 资产页持仓与钱包合并验收

> 范围：/portfolio 由占位「组合」重构为唯一「资产」承载页，合并仿真钱包、持仓明细、资金流水与资产统计；导航「组合」改名「资产」；「我的」(SettingsView) 移除钱包 tab。数量与成本来自后端仿真 projection（`GET /simulation/portfolio`），市值与浮动盈亏由前端用实时 PandaData 行情计算。

## 1. 任务与信息架构

| 验收项 | 结果 | 证据 |
| --- | --- | --- |
| 页面首屏能识别用户任务、当前对象和主要状态 | 通过 | kicker「资产 / ASSETS」+ 标题「持仓与资金」；左栏钱包、主区持仓明细/资金流水、右栏资产统计各自独立 |
| 每个区域只有一个明确职责 | 通过 | 钱包（余额/初始化/重置）、持仓明细、资金流水、资产统计四区职责单一 |
| 主操作唯一且结果可预测 | 通过 | 无账户时唯一主操作为「初始化仿真账户」；已有账户时为「刷新数据/重置钱包」，重置带二次确认 |
| 页面与上一步、下一步对象保持连续 | 通过 | 「我的」账户资料内保留「前往资产页」入口链接到 /portfolio；持仓标的并入共享 `DEFAULT_SYMBOLS` 轮询池 |
| 没有无功能价值的区域或卡片 | 通过 | 移除原 PortfolioView「暂不可计算」占位；各区均绑定真实数据或明确失败态 |

## 3. 真实数据

| 验收项 | 结果 | 证据 |
| --- | --- | --- |
| 市场数据来自 PandaData，不含硬编码行情 | 通过 | 现价取 `market.quotesMap.get(symbol).last`，无硬编码 |
| 显示来源、供应商时间、实际频率和新鲜度 | 不适用 | 资产页复用共享 market store，行情来源/时间在行情页展示，本页仅消费现价估值 |
| 页面刷新间隔与数据频率分别表达 | 通过 | 持仓标的经 `setWatchSymbols` 并入共享轮询，不独立轮询 |
| 请求失败不会回退到模拟行情 | 通过 | 缺行情标的显示「暂不可估值」（colspan=3），不以成交价伪造现价/盈亏；单测 `marks positions unvalued when PandaData quotes are unavailable` 断言 |
| 真实行情与仿真账户/订单明确分离 | 通过 | 数量/成本/已实现盈亏来自后端 `fetchPortfolio()`；市值/浮盈由前端行情乘数量计算，不混存 |
| 浏览器不可访问 PandaData 凭据 | 通过 | 前端仅访问同源 `/simulation/*`，无凭据 |

## 10. 最终结论

- 验收结论：通过。资产页持仓与钱包合并已落地，数量/成本来自后端 projection、市值/浮盈由前端实时行情计算、缺行情标的显式「暂不可估值」且不伪造盈亏；导航改名与「我的」钱包 tab 迁移完成。
- 未通过项目：无。
- 已知限制：
  - 本轮以单元测试与静态检查为主，未执行 1440/1024/<1024 px 浏览器实测与控制台核查，相关项标注为不适用（未执行）。
  - `npx vitest run` 在并行执行 + 多个 vite dev server 同时运行时出现跨文件测试隔离/计时抖动（core-behavior 轮询、layout-contract、trading-flow 间歇失败）；`--no-file-parallelism` 下 67/67 全部通过，属既有并行隔离问题，非本次改动引入。
- 复现命令或测试入口：
  - `cd frontend && npx vitest run --no-file-parallelism`（67 passed）
  - `cd frontend && npx vitest run src/tests/trading-flow.spec.ts src/tests/trading-failure-states.spec.ts`（9 passed）
  - `cd frontend && npx vue-tsc --noEmit`（无错误）
  - `cd frontend && npx eslint src --ext .ts,.vue`（0 errors）
