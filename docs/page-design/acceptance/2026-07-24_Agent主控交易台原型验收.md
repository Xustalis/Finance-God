# Agent 主控交易台隔离原型验收

> 验收对象：`frontend/prototypes/agent-workbench/`
> 验收日期：2026-07-24
> 结论口径：这是 Phase 1 隔离交互原型，不替代生产 T03 页面验收。

## 1. 任务与信息架构

| 验收项 | 结果 | 证据 |
| --- | --- | --- |
| 页面首屏能识别用户任务、当前对象和主要状态 | 通过 | `App.vue` 首屏显示交易台、当前 section/mode/标的、PandaData 状态和 Agent 上下文；`output/playwright/agent-workbench-1440.png` 可复现 |
| 每个区域只有一个明确职责 | 通过 | 左侧局部导航、中央当前任务、右侧 Agent、顶部提醒/用户设置边界各自独立 |
| 主操作唯一且结果可预测 | 通过 | 每个左侧工作态只有一个“运行当前分析/复核”主操作；Agent 输入属于常驻辅助操作 |
| 页面与上一步、下一步对象保持连续 | 通过 | section/mode 切换保留 `selectedSymbol`；Agent 上下文随左侧同步 |
| 没有无功能价值的区域或卡片 | 通过 | 页面只保留导航、工作区、状态、Agent、提醒和设置边界，没有装饰卡片 |

## 2. 文案

| 验收项 | 结果 | 证据 |
| --- | --- | --- |
| 标题直接说明区域内容或操作 | 通过 | 使用“仿真订单草稿”“当前仿真持仓”“提醒记录”等功能标题 |
| 辅助文字说明范围、操作、时效或决策价值 | 通过 | 辅助文字只解释仿真范围、行情来源、提交边界和恢复条件 |
| 删除了口号、隐喻、设计说明和重复文案 | 通过 | 页面无“智能中枢”等口号；原型边界使用事实标签表达 |
| 按钮使用“动词 + 对象/结果” | 通过 | “刷新行情”“运行订单草稿复核”“全部标为已读”等 |
| 错误、空状态和恢复条件表达明确 | 通过 | PandaData 失败显示错误码、“未使用演示价格替代”和刷新入口 |

## 3. 真实数据

| 验收项 | 结果 | 证据 |
| --- | --- | --- |
| 市场数据来自 PandaData，不含硬编码行情 | 通过 | `App.vue` 只请求 `/api/market/overview` 和 `/api/market/bars`；空响应直接报错 |
| 显示来源、供应商时间、实际频率和新鲜度 | 通过 | context bar、行情标题和图表元数据显示 provider、provider_time、frequency、freshness |
| 页面刷新间隔与数据频率分别表达 | 通过 | 顶部显示“请求间隔 60 秒”，工作区单独显示响应实际频率 |
| 请求失败不会回退到模拟行情 | 通过 | 本机 400/502 时价格保持“—”、K 线保持空并显示 `MARKET_DATA_INVALID_REQUEST` / `MARKET_DATA_DEADLINE_EXCEEDED` |
| 真实行情与仿真账户/订单明确分离 | 通过 | 行情区标“真实行情/PandaData”；持仓、订单、钱包和交易记录均标“仿真/结构示例” |
| 浏览器不可访问 PandaData 凭据 | 通过 | 浏览器只访问同源 `/api`；`vite.config.ts` 仅配置代理，原型没有凭据字段 |

## 4. 工作区与桌面适配

| 验收项 | 结果 | 证据 |
| --- | --- | --- |
| 1440 px 下信息层级和密度符合规范 | 通过 | `output/playwright/agent-workbench-1440.png` |

| 工作区布局可以保存、恢复和重置 | 失败 | Agent 收起偏好可通过 localStorage 恢复；左侧面板布局尚无保存、恢复和重置 |

## 6. 状态与交互

| 验收项 | 结果 | 证据 |
| --- | --- | --- |
| 加载、正常、空、延迟、过期和失败状态已覆盖 | 失败 | 代码覆盖加载、正常、空、freshness 和失败；本次真实链路只验证了失败，尚未复现实盘正常、延迟与过期 |
| 行情刷新不会清除选择或未提交输入 | 通过 | `loadMarketOverview/loadBars` 只更新行情状态，不修改 selectedSymbol、quantity 或 limit price |
| 隐藏标签页停止轮询，恢复后立即刷新 | 通过 | `onVisibilityChange` 在 hidden 时 `stopPolling`，恢复时 `refreshAll` 后 `startPolling` |
| 相同请求由统一控制器去重 | 通过 | overview 使用 `overviewInFlight`；bars 切换时取消旧请求 |
| 仿真订单状态未知时不能重复提交 | 通过 | 原型不注册 submit/cancel 动作，也不产生订单事实；动作合同测试覆盖危险动作拒绝 |

## 7. AI

| 验收项 | 结果 | 证据 |
| --- | --- | --- |
| AI 分析绑定当前对象和数据版本 | 通过 | 侧栏显示 section/mode/symbol 和 `workspace-context/v1`；工作流冻结输入步骤可见 |

## 10. 最终结论

- 验收结论：**作为 Phase 1 隔离交互原型有条件通过；不能作为生产 T03 完成声明。**
- 未通过项目：面板拖拽尺寸、工作区布局保存/重置、完整行情状态实测、Toast 焦点恢复、真实
  PandaData 冒烟。
- 已知限制：工作流、提醒、持仓和钱包是原型状态；没有真实模型、后台调度、SSE 或生产 Page View。
- 复现命令：

  ```bash
  cd frontend/prototypes/agent-workbench
  npm run test
  npm run type-check
  npm run lint
  npm run build
  npm run dev
  ```
