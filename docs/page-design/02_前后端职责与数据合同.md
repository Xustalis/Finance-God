# Finance-God 前后端职责与数据合同

> 文档类型：开发规范 / Reference  
> 适用范围：Finance-God 全部桌面端交易页面与共享前端组件  
> 目标读者：产品、前端、后端、测试、设计评审与实现 Agent  
> 规范状态：强制  
> 相关规范：[前端统一设计规范](01_前端统一设计规范.md)

## 1. 目标

本规范用于减少页面区域、功能入口、前端状态和浏览器端业务逻辑，使前端成为
清晰、稳定的任务界面，把聚合、计算、校验、权限、状态机和 AI 编排统一交给后端。

页面减肥不是隐藏复杂度，也不是删除风险信息。减肥后的页面仍须让用户明确知道：

1. 当前处理哪个账户、组合、标的、计划或订单；
2. 当前事实、版本、数据时点和新鲜度；
3. 当前唯一主要任务；
4. 哪些条件允许或阻止操作；
5. 操作提交后由哪个后端对象接管。

如果页面需要自行组合多个接口、重复计算业务结果或推断对象状态，说明复杂度尚未
真正下沉。

## 2. 适用边界

本规范负责定义前后端职责和页面复杂度预算。
[前端统一设计规范](01_前端统一设计规范.md)继续负责视觉、桌面宽度、
工作区、数据呈现、AI 侧栏和可访问性要求。

发生冲突时按以下顺序处理：

1. 安全、数据完整性、授权和订单不可重复提交规则最高优先；
2. 前端视觉与交互遵守 `01_前端统一设计规范.md`；
3. 前后端职责遵守本规范；
4. 页面专属文档只补充该路由的任务和字段，不得重新实现通用业务逻辑。

页面专属文档要求浏览器执行本规范已规定由后端承担的逻辑时，应在同一变更中更新
页面文档，不得保留两套实现。

## 3. 页面减肥原则

### 3.1 前端是任务终端

前端只完成四类工作：

- 展示后端返回的事实和可执行状态；
- 收集用户输入；
- 向后端发送明确命令；
- 展示命令结果、错误和后续状态。

前端不自行决定业务结果，不拥有授权、计划、订单、持仓、成交、复盘或 AI 产物的
事实状态。

### 3.2 一个页面只完成一个主要任务

每个路由必须定义一个页面任务和一个主要对象。与主要任务无直接关系的能力按顺序
处理：

1. 删除重复能力；
2. 合并为当前对象的详情；
3. 移到次级标签或只读 Inspector；
4. 深链到已有专属页面；
5. 由后端异步处理，页面只展示状态和结果。

不得为了“功能完整”把其他页面的任务复制到当前页面。

### 3.3 默认界面只展示决策所需信息

专业数据不等于默认全部展开。默认页面只展示完成当前任务所需的字段；证据、版本、
计算明细和诊断信息按需进入 Inspector、T10 或专属详情。

以下内容不得占据默认工作区：

- 已在全局栏持续展示的账户、连接和刷新信息；
- 与当前对象无关的市场、新闻或系统状态；
- 后端计算过程的中间变量；
- 重复的 AI 摘要、规则摘要和对象摘要；
- 仅用于证明系统“能力丰富”的模块；
- 没有独立操作价值的装饰性统计。

### 3.4 只保留一个事实源

浏览器可以缓存查询结果和未提交表单，但不能把缓存变成新的领域事实源。

- 订单状态来自执行服务；
- 持仓变化来自 `Fill` 和组合服务；
- 授权状态来自授权服务；
- 计划版本来自交易计划服务；
- 行情来自 PandaData 后端适配器；
- AI 结论和证据来自研究产物存储；
- 页面可用操作来自后端能力判断。

刷新、切换页面或重新连接后，前端必须重新接受后端事实，不做本地状态合并推断。

### 3.5 失败必须可见

页面减肥不能通过默认值、模拟结果、静默重试或隐藏错误来实现。

- 请求失败保留最后成功值并显示过期状态；
- 计算失败显示失败步骤、影响范围和 Trace ID；
- 状态未知时只允许查询，不允许重复提交；
- 缺少业务结果时显示缺失，不在浏览器补算；
- 后端未返回操作权限时不猜测为允许。

## 4. 页面复杂度预算

新页面和改造后的页面默认满足以下预算：

| 项目 | 预算 | 说明 |
| --- | ---: | --- |
| 页面主要任务 | 1 个 | 可用一句动宾短语描述 |
| 页面主要对象 | 1 个 | 列表页可将“当前查询结果”作为主要对象 |
| 视觉主操作 | 1 个 | 其他操作降为次级、行内或溢出菜单 |
| 页面级标签栏 | 最多 1 行 | 不在面板内继续嵌套同级标签 |
| 根级工作区区域 | 最多 3 个 | 不含产品壳和共享 AI 侧栏 |
| 页面纵向滚动轴 | 1 个 | 表格内部虚拟滚动除外 |
| 嵌套抽屉 | 0 个 | 抽屉内需要更多任务时进入独立路由 |
| 默认摘要指标 | 最多 5 个 | 仅保留会改变判断或操作的指标 |
| 页面自建领域状态机 | 0 个 | 领域状态由后端返回 |
| 页面重复业务计算 | 0 个 | 金额、风险、权限和状态不在组件中重算 |
| 页面组件独立行情轮询 | 0 个 | 统一控制器请求标准化接口 |

超出预算必须在页面专属设计简报中说明：

- 为什么不能删除或深链；
- 该复杂度服务哪个主要任务；
- 后端已承担哪些计算和状态；
- 1024 px 下如何保持主任务完整。

“专业用户需要”不能单独作为超出预算的理由。

## 5. 前后端职责

### 5.1 前端保留

| 职责 | 前端可做 | 限制 |
| --- | --- | --- |
| 页面结构 | 导航、工作区、表格、图表、Inspector、共享 AI 侧栏 | 不从结构推断业务状态 |
| 展示格式 | 单位、精度、日期、时区、颜色和可访问标签 | 原始数值和精度规则由合同提供 |
| 临时交互状态 | 选中行、展开项、排序显示、未提交输入 | 不跨对象继承业务值 |
| 工作区偏好 | 面板显隐、宽度、标签顺序、AI 侧栏状态 | 仅属于当前用户的界面偏好 |
| 输入检查 | 必填、格式、长度、数值可解析性 | 不代替授权、资金、风控和交易校验 |
| 命令提交 | 发送明确命令和幂等键 | 不直接修改查询缓存伪造成功 |
| 状态反馈 | 加载、空、延迟、过期、失败、提交中 | 以后端状态码和版本为准 |
| 图表绘制 | 绘制后端提供的行情、指标和图层数据 | 不在页面组件内计算交易指标 |
| 可访问性 | 键盘、焦点、语义、文字摘要、reduced motion | 不因精简而删除必要语义 |

### 5.2 后端必须承担

| 职责 | 后端输出 |
| --- | --- |
| 数据接入 | PandaData 凭据管理、请求、缓存、批量、去重和标准化 |
| 页面聚合 | 当前任务所需的统一 View DTO，不要求浏览器拼接多个领域对象 |
| 领域计算 | 估值、收益、费用、滑点、风险、偏离、集中度、重叠和压力情景 |
| 状态机 | 授权、计划、订单、成交、复盘、Agent Run 的合法状态与迁移 |
| 业务校验 | 账户、权限、授权、余额、数量、价格、市场、数据版本和硬风控 |
| 能力判断 | 当前可执行操作、禁用原因、恢复条件和所需确认 |
| 幂等与并发 | 幂等键、版本检查、重复提交保护、乐观并发和对账 |
| 搜索与筛选 | 大数据集的查询、分页、排序、组合筛选和权限裁剪 |
| AI 编排 | 上下文装配、Prompt、模型调用、工具调用、证据关联和产物版本 |
| 审计与证据 | 来源、版本、规则命中、用户确认、执行事件和 Trace ID |
| 错误分类 | 稳定错误码、用户可读信息、影响范围和可恢复动作 |

### 5.3 明确禁止放在前端的逻辑

- 根据多个接口结果推断订单最终状态；
- 根据成交记录直接修改持仓；
- 根据当前输入自行判断授权或硬风控通过；
- 在不同组件中分别计算市值、费用、风险或组合偏离；
- 在浏览器装配完整 AI 交易上下文或保存系统 Prompt；
- 用按钮是否显示代替后端权限校验；
- 用本地时间判断行情、计划、授权或订单是否仍有效；
- 失败后切换到硬编码行情或演示业务数据；
- 通过更新查询缓存把“请求已发送”显示为“已受理”；
- 为同一领域对象维护 Redux、Context、组件状态和查询缓存等多份可写副本。

## 6. 标准化后端合同

### 6.1 页面查询

后端应为每个路由提供任务导向的 View DTO。DTO 可以按路由定义，但使用统一外层：

```ts
type PageView<TObject, TData> = {
  object: TObject;
  data: TData;
  version: string;
  generatedAt: string;
  dataStatus: {
    provider: string;
    providerTime: string | null;
    frequency: string | null;
    freshness: "fresh" | "delayed" | "stale" | "unknown";
    lastSuccessAt: string | null;
  };
  capabilities: Array<{
    action: string;
    enabled: boolean;
    reasonCode: string | null;
    reason: string | null;
  }>;
  warnings: Array<{
    code: string;
    severity: "info" | "warning" | "blocking";
    message: string;
    affectedFields: string[];
  }>;
};
```

要求：

- View DTO 直接支持页面任务，不暴露浏览器无需理解的内部实体图；
- 数值结果同时返回单位、币种、精度或格式规则；
- `null` 表示不可用，不能用 `0` 或空字符串代替；
- 数据、能力和警告使用同一对象版本生成；
- `capabilities` 用于呈现操作状态，但命令提交时后端仍须重新校验；
- 大型时间序列可使用独立端点或流，不得塞入页面首屏 DTO。

### 6.2 命令

前端发送命令，不直接提交整份领域对象：

```ts
type Command<TPayload> = {
  objectId: string;
  expectedVersion: string;
  idempotencyKey: string;
  payload: TPayload;
};
```

命令响应至少返回：

- 命令是否接收；
- 新对象版本或异步任务 ID；
- 当前确定状态；
- 下一次允许查询的对象；
- 错误码、阻断原因和可恢复动作。

对于订单，`accepted: true` 只表示命令已被系统接收，不能等同于订单已受理或已成交。

### 6.3 错误

统一错误结构必须包含：

```ts
type ProblemDetails = {
  code: string;
  message: string;
  traceId: string;
  retryable: boolean;
  affectedObject: {
    type: string;
    id: string;
    version: string | null;
  } | null;
  recoveryAction: string | null;
};
```

前端根据 `code` 选择固定呈现方式，不解析自然语言来决定业务行为。

## 7. 数据与刷新减肥

### 7.1 页面只订阅所需数据

- 产品壳读取账户、PandaData、AI 和刷新状态；
- 当前路由读取一个主要 Page View；
- 图表或超大表格按需读取专属数据；
- Inspector 在选中对象后加载详情；
- 隐藏面板不发起专属请求；
- 同一对象的相同请求由共享查询层去重。

### 7.2 统一轮询

浏览器可保留一个共享轮询调度器，用于按用户选择的间隔请求后端标准化接口。
PandaData 上游请求、缓存、批量和频率控制属于后端职责。

页面组件不得：

- 各自建立 `setInterval`；
- 直接请求 PandaData；
- 根据轮询间隔推断数据频率；
- 在刷新时重置选择、滚动位置或未提交表单；
- 在页面隐藏时继续发起行情请求。

### 7.3 派生数据

只允许前端做不改变业务含义的展示派生，例如：

- 将秒转换为“3 分钟前”；
- 按合同给定精度格式化金额；
- 根据当前选择过滤已经完整加载的小型展示列表；
- 从现有字段生成无业务判断的可访问性文本。

只要结果会影响交易、风险、权限、排序、候选或 AI 结论，就必须由后端计算并带版本
返回。

## 8. 功能保留、合并与删除

评审每个页面功能时按以下顺序判断：

### 8.1 保留

同时满足以下条件才在默认页面保留：

- 直接服务当前页面主要任务；
- 用户需要在当前上下文中查看或操作；
- 无法由已有字段自然表达；
- 在 1024 px 下仍能保持可用；
- 不要求浏览器拥有新的领域状态或业务计算。

### 8.2 合并

出现以下情况时合并：

- 多个摘要描述同一个对象；
- 多个状态条来源相同；
- 图表、表格和 Inspector 重复展示同一字段；
- 多个按钮最终进入同一个下游任务；
- AI 解释与普通说明重复。

合并后保留一个权威位置，其他区域通过选中、深链或锚点定位。

### 8.3 深链

功能属于其他页面主要任务时使用深链：

- 授权与账户管理进入 T00；
- 标的筛选和候选进入 T02；
- 单标的研究与订单草稿进入 T03；
- 计划审阅进入 T04；
- 订单事实与异常进入 T07；
- 组合和偏离进入 T08；
- 复盘进入 T09；
- 证据、版本和诊断进入 T10。

### 8.4 删除

满足任一条件应删除：

- 没有明确用户任务；
- 只是重复已有事实；
- 只是展示系统能力或设计概念；
- 依赖前端推断业务结果；
- 用户不能据此做出不同判断或操作；
- 与主要任务竞争注意力；
- 失败后只能依靠模拟值维持展示。

## 9. 各页面减肥方向

| 页面 | 默认保留 | 下沉后端 | 删除、合并或深链 |
| --- | --- | --- | --- |
| T00 设置 | 账户选择、授权摘要、授权编辑、影响结果、保存新版本 | 连接状态、权限裁剪、规则冲突、影响分析、版本创建 | 安全诊断和完整历史进入 T10；不在前端计算授权影响 |
| T01 总览 | 市场状态、组合摘要、按优先级排列的待办、最近活动 | 跨服务聚合、P0–P3 排序、去重、已读和失效判断 | 删除独立风险墙和行情机会区；每个待办只保留一个深链动作 |
| T02 行情 | 搜索、一个主列表、筛选、单一 Inspector、进入交易台 | 候选生成、排除、排序、评分维度、组合影响预览 | 自选与候选用互斥标签；高级比较进入全宽状态；不展示神秘总分 |
| T03 交易台 | 当前标的、图表、持仓/订单摘要、研究或订单单一工作态 | 指标、策略、估算、费用、风险校验、计划偏离和 AI 上下文 | 研究态与订单态互斥；候选管理进入 T02；全账户订单进入 T07 |
| T04 计划 | 调整原因、动作列表、组合影响、阻断、确认并生成草稿 | 当前/目标/计划后计算、成本、风险、动作依赖、版本和校验 | 证据明细进入 T10；单标的深度研究进入 T03；不内嵌执行中心 |
| T07 订单 | 状态筛选、订单表、选中订单 Inspector、合法撤单操作 | 订单状态机、事件对账、批次依赖、成交汇总、异常恢复 | 组合详情进入 T08；计划详情进入 T04；状态未知不提供重提入口 |
| T08 组合 | 组合摘要、配置对比、持仓表、单一偏离 Inspector、创建计划 | 估值、收益、偏离、风险、重叠、压力情景、同步冲突 | 单标的研究进入 T03；调仓动作进入 T04；来源明细进入 T10 |
| T09 复盘 | 计划/修改/执行比较、关键偏差、用户反馈、保存复盘 | 对象对齐、执行指标、费用、组合前后变化、证据完整度 | 原始事件进入 T07/T10；不在前端计算归因或用短期盈亏评分 |
| T10 证据 | 对象定位、结论、事实/推断/未知、版本和事件只读视图 | 血缘图、权限裁剪、版本比较、Agent 运行和证据包生成 | 普通模式隐藏内部栈和 Prompt；需要复杂比较时进入独立高级视图 |

T05 订单草稿和 T06 最终复核继续作为 T03 内部逻辑区域，不新增一级路由：

- T05 只编辑后端草稿 DTO；
- T06 只展示不可编辑的最终摘要并发送确认命令；
- T06 不重新计算金额、费用、风险或权限；
- 任一版本变化都要求后端返回新的复核快照。

## 10. AI 侧栏减肥

共享 AI 侧栏只接收：

- 当前路由；
- 当前对象类型、ID 和版本；
- 当前用户问题；
- 必要的显示偏好。

后端负责装配市场、组合、计划、授权、订单和证据上下文。浏览器不得：

- 拼接完整领域对象作为 Prompt；
- 保存系统 Prompt、工具定义或模型凭据；
- 在多个页面分别实现 AI 请求流程；
- 将流式文本当作结构化交易结论；
- 在没有产物版本时允许填入订单草稿。

后端返回的 AI 产物至少包含：

- 产物 ID 和版本；
- 当前对象和数据版本；
- 结构化结论；
- 支持证据、反方证据和未知项；
- 适用周期和失效条件；
- 允许的后续动作。

页面只负责流式呈现、选择证据、提交追问和触发后端声明允许的动作。

## 11. 表单与操作减肥

### 11.1 表单

- 默认只显示完成命令所需字段；
- 高级参数使用一个“高级设置”区域，不分散到多个卡片；
- 后端提供默认值、范围、步长、单位和依赖关系；
- 前端只做格式和必填检查；
- 业务错误显示在对应字段和页面级阻断区；
- 行情刷新不得覆盖用户输入。

### 11.2 操作

页面主操作由后端 `capabilities` 决定启用状态。次级操作按以下顺序收敛：

1. 当前对象必要的行内操作；
2. 低频操作进入溢出菜单；
3. 跨任务操作改为深链；
4. 诊断和证据进入 T10；
5. 无法说明结果的操作删除。

按钮禁用时必须显示后端返回的原因和恢复条件。隐藏按钮不能替代权限控制。

## 12. 实施顺序

页面减肥按以下顺序执行：

1. 为页面写明主要任务、主要对象和唯一主操作；
2. 盘点页面区域、接口、状态、计算和命令；
3. 标记删除、合并、深链、前端保留和后端下沉；
4. 先实现后端 View DTO、命令和错误合同；
5. 删除前端重复计算、拼接和领域状态机；
6. 将页面收敛为一个主要查询、必要的按需详情和命令；
7. 校验加载、空、延迟、过期、失败和并发冲突；
8. 在 1440 px 和 1024 px 验证任务完整性；
9. 使用前端设计验收模板逐项报告。

不得先隐藏旧功能，再保留旧逻辑和接口。减肥必须同时删除不再使用的组件、状态、
请求、计算、测试和文档说明。

## 13. 代码评审检查

### 13.1 页面

- [ ] 页面能用一句话说明主要任务；
- [ ] 默认工作区不超过三个根级区域；
- [ ] 只有一个视觉主操作；
- [ ] 没有重复摘要、重复状态条或重复对象详情；
- [ ] 跨任务功能已改为深链；
- [ ] 1024 px 下不靠隐藏风险和主操作维持布局；
- [ ] 没有嵌套抽屉和互相竞争的纵向滚动。

### 13.2 状态与数据

- [ ] 页面不拥有领域状态机；
- [ ] 页面不拼接多个接口推断业务结果；
- [ ] 页面不计算影响交易判断的派生值；
- [ ] 同一事实只有一个可写来源；
- [ ] 行情请求由共享控制器发起；
- [ ] 失败、过期、状态未知和版本冲突均显式呈现；
- [ ] 真实行情和仿真业务数据保持分离。

### 13.3 命令与安全

- [ ] 所有写操作使用明确命令、对象版本和幂等键；
- [ ] 后端在提交时重新校验权限和业务规则；
- [ ] 前端没有 PandaData、经纪商或模型凭据；
- [ ] 查询缓存不会伪造命令成功；
- [ ] 订单提交、受理、成交和撤单状态没有合并；
- [ ] 日志和错误没有暴露敏感字段。

### 13.4 删除完整性

- [ ] 被删除功能的组件、状态、请求和测试已同时处理；
- [ ] 没有保留第二套计算、校验或权限逻辑；
- [ ] 没有用通用 `try/catch`、默认值或模拟数据掩盖失败；
- [ ] 页面专属文档已同步更新；
- [ ] 没有未说明的行为变化。

## 14. 验收条件

页面减肥完成必须提供以下证据：

- 改造前后区域、功能入口和浏览器请求数量对比；
- 已删除或下沉逻辑清单；
- 后端 View DTO、命令和错误合同；
- 目标页面的加载、空、过期、失败和版本冲突测试；
- 领域计算和状态机的后端单元测试；
- 前端不再包含重复业务计算的代码检索结果；
- 1440 px、1024 px 和低于 1024 px 的浏览器验证；
- 按
  [前端设计验收模板](templates/前端设计验收模板.md)
  填写的逐项结果。

只删除可见控件、把功能塞进更多菜单或把代码移到另一个前端组件，不视为完成页面
减肥。

## 15. 禁止事项

- 用卡片折叠代替信息架构精简；
- 用更多标签页容纳未裁剪功能；
- 把所有后端字段一次性展示给专业用户；
- 在浏览器复制后端计算以获得“即时反馈”；
- 为按钮状态建立第二套前端权限逻辑；
- 用乐观更新伪造订单、成交、授权或持仓事实；
- 将复杂逻辑移入前端共享工具后宣称已经下沉；
- 用 AI 文本代替确定性计算、校验或错误码；
- 因页面减肥删除数据来源、时点、风险、错误和可访问性信息；
- 保留无人使用的兼容分支和隐藏功能入口。
