# Finance-God 前端设计要求

> 文档类型：规范 / Reference  
> 适用范围：Finance-God 全部桌面端交易页面  
> 目标读者：负责产品设计、前端实现、评审与验收的 Agent 和开发者  
> 规范状态：强制；与旧页面文档冲突时以本文件为准

## 1. 使用目标

本规范用于确保 Agent 生成的是可操作的专业交易软件，而不是由卡片、口号和
概念说明组成的演示页面。

一个合格页面必须让用户在第一次扫描时回答五个问题：

1. 当前处理的账户、组合、标的、计划或订单是什么；
2. 数据来自哪里，更新到什么时间，频率是多少；
3. 当前状态是否正常，异常会影响什么；
4. 用户现在可以执行什么操作；
5. 操作后会改变哪个仿真对象。

如果标题、标签、数字和操作不足以回答这些问题，应修改信息结构，而不是增加
一段营销式说明。

## 2. 产品与运行边界

### 2.1 用户

Finance-God 同时服务具备投资基础的个人用户和专业交易者。两类用户共用一套
工作区：

- 默认布局呈现完成任务所需的最小信息；
- 专业数据、指标和工具通过标签或可配置面板打开；
- 不增加独立“新手模式”，也不通过隐藏关键风险简化界面；
- 术语可以专业，但必须保持一致并提供明确单位、范围和时点。

### 2.2 设备

- 主要设计宽度：1440 px；
- 最低支持宽度：1024 px；
- 1024–1279 px：压缩工具栏、允许面板折叠、保持主要表格可用；
- 小于 1024 px：显示桌面端访问提示，不生成移动页面；
- 不提供移动底部导航、移动复核流程或按比例缩小的桌面页面。

### 2.3 数据与交易

- PandaData 是市场行情、资产主数据、基本面、因子、资金、事件和宏观数据的
  唯一前端来源；
- 市场数据是真实数据；
- 账户、持仓、计划、订单、成交和复盘是仿真数据；
- 页面最高支持每秒轮询一次，但不得将轮询频率描述为数据频率；
- 每个市场数据区域必须显示来源、供应商时间、实际频率和新鲜度；
- 请求失败、空响应或字段不完整时显示错误，不用演示数据替换；
- 公募基金净值、港美股日线等低频数据必须明确标注“日频”或对应频率。

### 2.4 生产技术基线

OKX 的工程拆解只作为架构分层和交互效率参考，不复制其内部 OKFE/`okui-` 组件、
CSS、品牌资产或页面实现。Finance-God 生产前端采用：

- Next.js 13+ App Router 与 React 19；
- 产品壳、稳定页面结构和首屏数据使用 SSR；图表、轮询、布局拖动、表单和模态层
  使用边界清晰的 Client Components；
- 建立 Finance-God 自有组件层和设计 Token，不直接使用 `okui-` 命名；
- 样式按 Token、基础组件、工作区和页面模块分层；允许少量工具类，但不得形成
  无归属的超大 CSS 包；
- 生产交易图表使用 `lightweight-charts`，图表外提供等价的文字摘要；
- PandaData 通过服务端适配器和标准化 API 提供，凭据不进入浏览器；
- 生产错误和性能监控可接入 Sentry，但不得上报 PandaData 凭据、订单输入或其他
  敏感字段。

生产前端必须严格遵循上述基线，不使用静态原型或非生产技术。
OKX 首页的视频营销、深色主题、移动优先布局、GTM 和同意管理不是本交易工作区的
默认需求。

## 3. 设计命题

### 3.1 视觉命题

**一套浅色、紧凑、数据优先的专业交易工作区：白色画布、细分隔线、近黑文字，
以蓝色标识选择和主要操作，以红绿表达交易与损益状态。**

参考 OKX 的工作区密度、模块化面板和操作效率，但不复制其商标、品牌资产、
图标、文案或页面像素结构。

### 3.2 交互命题

1. 当前标的、数据时点和账户模式在跨页面操作中保持连续；
2. 表格选择、图表、Inspector 和常驻 AI 侧栏共享同一当前对象；
3. 面板可以显示、隐藏、调整宽度和恢复默认布局；
4. 刷新只更新数据，不改变选择、滚动位置或未提交的表单；
5. 一个区域只提供一个明确的首要动作。

## 4. 信息架构

### 4.1 顶部产品导航

一级导航固定为：

| 导航 | 页面 | 用户任务 |
| --- | --- | --- |
| 总览 | T01 | 查看市场、组合状态与需要处理的任务 |
| 行情 | T02 | 搜索、筛选、比较和跟踪全部可用标的 |
| 交易台 | T03；包含 T05/T06 | 分析当前标的并创建仿真订单 |
| 组合 | T08；T04 为计划详情 | 查看配置、偏离和调仓计划 |
| 订单 | T07 | 跟踪仿真订单、成交与异常 |
| 复盘 | T09 | 评价计划、执行与结果 |
| 数据 | T10 | 查看 PandaData 数据、证据与 Agent 运行 |
| 设置 | T00 | 管理仿真账户、授权和工作区 |

一级导航使用文字，不使用只有图标的 80 px 左侧栏。当前项由文字权重、底线和
`aria-current` 同时表达。

### 4.2 工作区标签

页面存在多个工具时，使用第二层工作区标签。例如：

- 行情：自选、全市场、筛选器、事件；
- 交易台：图表、盘口、研究、订单草稿；
- 组合：持仓、配置、偏离、调仓计划；
- 数据：行情、基本面、资金、因子、衍生品、宏观、Agent 证据。

工作区标签可以排序和关闭，但核心标签必须能从“添加面板”恢复。

### 4.3 全局状态区

顶部右侧持续显示：

- 仿真账户标识；
- PandaData 连接状态；
- 最近成功更新时间；
- 当前刷新间隔；
- AI 任务状态；
- AI 侧栏展开或收起状态；
- 工作区布局菜单。

不使用“全局事实条”等设计说明作为界面标题。每个值直接使用产品语言。

## 5. 页面骨架

```text
┌───────────────────────────────────────────────────────────────────────────┐
│ 产品标识 │ 总览 行情 交易台 组合 订单 复盘 数据 设置 │ 搜索 │ 状态 │ 账户 │
├───────────────────────────────────────────────────────────────────────────┤
│ 页面/标的标题 │ 工作区标签                    │ 刷新 │ 布局 │ 页面操作 │
├──────────────┬──────────────────────┬──────────────┬──────────────────────┤
│ 可选列表     │ 主任务工作区         │ Inspector    │ 常驻、可收起 AI 侧栏 │
│ 页面可配置区域；允许隐藏、调宽和排序                  │ 当前页面/对象上下文  │
└──────────────┴──────────────────────┴──────────────┴──────────────────────┘
```

不是每个页面都需要三栏。区域只有在承担独立浏览或操作职责时才成立。

## 6. 视觉系统

### 6.1 色彩

| Token | 建议值 | 用途 |
| --- | --- | --- |
| `--canvas` | `#F5F6F8` | 应用背景与区域分隔 |
| `--surface` | `#FFFFFF` | 工作区 |
| `--surface-subtle` | `#F8F9FB` | 次级工具条、表头 |
| `--line` | `#E3E6EA` | 常规分隔线 |
| `--line-strong` | `#C9CED6` | 面板边界、拖动手柄 |
| `--text` | `#111318` | 主文本 |
| `--text-secondary` | `#5E6570` | 次要事实 |
| `--text-tertiary` | `#8B929C` | 时间、来源、占位 |
| `--accent` | `#1769E0` | 选中和主要操作 |
| `--positive` | `#0B8F55` | 上涨、买入、通过 |
| `--negative` | `#D9363E` | 下跌、卖出、失败 |
| `--warning` | `#B26A00` | 延迟、待确认 |

规则：

- 同一区域只出现一个主要强调色；
- 蓝色不能用于普通边框、装饰图形或大面积背景；
- 红绿必须同时配合符号、文字或图标；
- 不使用装饰渐变、玻璃拟态和常规卡片阴影；
- 浮层可以使用轻阴影区分层级。

### 6.2 字体与数字

- 中文：系统无衬线，优先 `PingFang SC`；
- 拉丁、数字和代码：系统字体，启用 `font-variant-numeric: tabular-nums`；
- 产品标识：18 px / 700；
- 页面标题：20 px / 650；
- 区域标题：14 px / 650；
- 正文和表格：13 px；
- 辅助信息：12 px；
- 价格、金额、数量右对齐；
- 货币符号、千分位、小数精度和涨跌符号必须统一。

### 6.3 密度

- 顶部导航：48 px；
- 页面工具栏：44 px；
- 工作区标签：36 px；
- 常规表格行：38 px；
- 紧凑表格行：32 px；
- 输入框与按钮：32–36 px；
- 页面外边距：12 px；
- 面板内边距：12–16 px；
- 面板圆角：0–6 px，默认通过分隔线而不是卡片表达区域。

## 7. 文案规范

### 7.1 标题

标题说明区域是什么或用户可以做什么：

- 正确：`最近活动`、`当前委托`、`组合偏离`、`数据质量`；

### 7.2 辅助文字

辅助文字只允许完成以下一项任务：

1. **范围**：包含哪些对象或时间；
2. **操作**：选择、筛选或点击后会发生什么；
3. **时效**：数据更新频率、时间或延迟；
4. **决策价值**：该数据会影响哪个判断。

示例：

| 情况 | 不合格 | 合格 |
| --- | --- | --- |
| 最近活动 | 授权、计划、订单、成交和复盘共用一条事实流 | 按时间展示授权变更、计划、订单、成交和复盘记录；选择记录可查看详情 |
| 行情 | 显示市场状态和数据新鲜度 | PandaData A 股快照；当前工作区每 1 秒请求一次 |
| AI | AI 为你的决策保驾护航 | 基于当前标的、组合和数据版本生成分析；结论不修改订单 |
| 组合 | 一站式掌握财富全景 | 显示仿真持仓、目标权重和计划后权重 |

如果删除辅助文字不影响理解，应删除。

### 7.3 按钮

按钮使用“动词 + 对象/结果”：

- `查看订单详情`
- `创建调仓计划`
- `展开 AI 侧栏`
- `收起 AI 侧栏`
- `保存工作区`

禁止使用无法预测结果的 `确定`、`处理`、`立即体验` 或 `开始探索`。

### 7.4 数据标签

- “实时”只用于供应商明确提供实时快照的数据；
- “1 秒刷新”描述页面请求间隔，不描述行情粒度；
- “仿真”必须出现在账户、订单、持仓和成交附近；
- “暂无数据”不能代替错误、权限不足或市场休市；
- 数据异常必须说明恢复条件或下一步操作。

## 8. PandaData 呈现与刷新

### 8.1 统一数据合同

前端行情对象至少包含：

```text
symbol, name, asset_type, market, currency,
last, open, high, low, previous_close,
change, change_percent, volume, amount,
provider, provider_time, frequency, freshness, market_status
```

字段不可用时返回 `null` 并显示 `—`，不得计算或编造替代值。

### 8.2 刷新策略

- 用户可选：1 秒、3 秒、5 秒、15 秒、60 秒、暂停；
- 当前可见页面共享一个轮询控制器；
- 浏览器隐藏时停止轮询；
- 恢复可见时立即请求一次；
- 后端对相同标的批量请求、缓存和去重；
- 请求进行中不启动下一次相同请求；
- 失败后保留最后成功值，同时显示“已过期”和失败时间；
- 不静默切换到其他数据供应商。

### 8.3 数据频率

刷新状态必须区分：

- 页面请求间隔；
- PandaData 数据集频率；
- 供应商记录时间；
- 当前市场状态；
- 距最后成功请求的时间。

## 9. AI 设计

AI 不作为独立聊天产品覆盖页面。它是所有桌面页面的常驻右侧栏。

### 9.1 上下文入口

- 行情：解释当前异动、指标和事件；
- 交易台：总结支持证据、反方证据、未知项和失效条件；
- 组合：解释偏离、集中度和计划影响；
- 订单：解释仿真执行状态，不猜测未知状态；
- 复盘：比较计划、行为和结果。

### 9.2 统一常驻 AI 侧栏

所有 T00–T10 页面都渲染同一个右侧 AI 侧栏。侧栏显示：

- 当前对象及版本；
- 使用的数据来源和时点；
- 结论；
- 证据和反方证据；
- 未知项；
- 可继续追问的输入框。

行为规则：

- 1280 px 及以上默认展开；用户可收起为 44 px 可见窄栏；
- 1024–1279 px 默认收起，但用户可以主动展开；
- 收起状态、当前页面和当前标的跨路由保持连续，并保存到当前浏览器；
- 页面切换、表格选择和标的搜索后，侧栏上下文同步更新；
- 收起不停止行情刷新，也不清除未提交的追问文本；
- 侧栏是应用骨架的一部分，不使用遮罩，不抢占模态焦点，也不覆盖订单操作；
- T06 最终复核打开时，AI 侧栏不可代替确认或修改复核内容；
- AI 不得自动创建、修改或提交仿真订单。

## 10. 工作区配置

相关页面必须支持：

- 面板显示/隐藏；
- 左、中、右区域拖动调整；
- 标签排序；
- 保存当前布局；
- 恢复页面默认布局；
- 使用浏览器存储恢复布局。

调整时设置最小宽度，禁止把订单表单、价格或主要操作压缩到不可用。布局设置
失败时显示错误，不假装已保存。

## 11. 页面要求

### T00 设置

- 工作区：仿真账户、授权、数据连接、AI 与布局；
- 显示 PandaData 连接、最后请求和凭据是否配置，不显示凭据内容；
- 主操作：`保存设置`；
- 布局重置与刷新频率独立保存。

### T01 总览

- 首屏顺序：市场状态 → 仿真组合 → 待处理任务 → 最近活动；
- 市场概览显示真实 PandaData 数据；
- 组合和任务明确标记为仿真；
- 最近活动说明固定为功能语言；
- 主操作深链到最高优先级任务。

### T02 行情

- 支持全部 PandaData 标的类型和数据模块；
- 左侧搜索/列表，中部表格或图表，右侧标的 Inspector；
- 表格显示供应商时间和实际频率；
- 主操作：`在交易台打开`。

### T03 交易台

- 默认区域：标的列表 / 图表与盘口 / 仿真订单与 AI；
- 图表、订单和持仓共享当前标的；
- 行情为真实，账户和订单为仿真；
- 订单参数不得因行情刷新而重置；
- T06 复核仍是提交仿真订单的唯一入口。

### T04 调仓计划

- 显示仿真计划目标、动作、影响和质量检查；
- 行情估算使用真实 PandaData 最新值并显示时点；
- 主操作：`确认计划并生成草稿`。

### T07 订单

- 显示仿真订单、成交和事件；
- 明确区分提交中、状态未知、已受理、部分成交和已成交；
- 状态未知只允许查询，不允许重复提交。

### T08 组合

- 显示仿真持仓；
- 市价和市值估算使用真实 PandaData 行情；
- 支持当前、目标和计划后三组权重；
- 主操作：`创建调仓计划`。

### T09 复盘

- 顺序：计划遵守 → 执行偏差 → 仿真结果 → AI 解释 → 用户结论；
- 不用单一收益数字评价行为质量；
- 主操作：`完成本次复盘`。

### T10 数据

- 使用标签组织 PandaData 行情、参考、基本面、资金、因子、期货、期权和宏观数据；
- 显示请求参数、响应行数、时间、频率和错误；
- Agent 证据与市场数据分区；
- 导出为次级操作。

## 12. 状态要求

每个数据区域必须覆盖：

- 首次加载；
- 正常；
- 空结果；
- 休市；
- 延迟；
- 过期；
- 权限不足；
- 请求失败；
- 字段缺失。

每个交互区域必须覆盖：

- 默认；
- 悬停；
- 焦点；
- 禁用；
- 提交中；
- 成功；
- 失败。

状态依赖文字、图标和形状共同表达，不能只依靠颜色。

## 13. 可访问性

- 所有功能可通过键盘完成；
- 使用原生按钮、输入、表格、对话框和标签语义；
- 焦点环持续可见；
- 模态层和抽屉打开后焦点进入，关闭后恢复；
- 图表提供文字摘要；
- 拖动面板同时提供键盘或菜单调整方式；
- 动效遵守 `prefers-reduced-motion`；
- 文本和交互控件满足 WCAG AA 对比度。

## 14. Agent 强制流程

### 14.1 设计前

Agent 必须先输出：

1. 页面用户与任务；
2. 当前对象和数据来源；
3. 页面区域及其职责；
4. 主操作和启用条件；
5. 需要覆盖的状态；
6. 桌面宽度与面板行为；
7. 与既有页面的连续关系。

### 14.2 实现中

- 先建立页面骨架和真实数据合同；
- 再实现核心任务；
- 然后添加 AI、布局配置和次级工具；

## 15. 拒绝条件

出现任一项即不能通过设计验收：

- 页面使用营销口号或设计解释代替功能说明；
- 常规页面由大量独立圆角卡片组成；
- 市场行情来自硬编码演示值；
- 失败时静默显示模拟行情；
- 把 1 秒轮询描述成 1 秒行情；
- 真实行情和仿真账户数据没有明显区分；
- 小于 1024 px 时生成未经设计的移动布局；
- 多个组件各自轮询相同标的；
- 蓝色用于大面积装饰；
- AI 结论没有当前对象、数据时点或证据；
- 页面只能使用鼠标操作；
- Agent 未提交逐项验收结果。
