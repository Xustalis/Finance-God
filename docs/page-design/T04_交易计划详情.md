# T04 交易计划详情页面

## 页面定位

当前页面定位为 **从组合目标到一组可审阅交易动作的解释与决策页**。

页面先回答“为什么调整、调整什么、不调整会怎样”，再展示订单草稿。它支持单标的计划和多标的调仓计划，但不直接伪装成已提交订单。

最终版的状态跃迁为：用户触发唯一主操作“确认计划并生成草稿”后，计划进入已确认状态并创建 `OrderDraft`；最终确认与 `OrderIntent` 创建仍只发生在 T06。

## 与 AI 辅助交易台的关系

- 单标的计划可进入 AI 辅助交易台继续研究和编辑订单草稿。
- 多标的计划可逐笔打开交易台，也可在本页生成批次草稿并逐项复核。
- 交易台对计划参数的修改必须回写为新的 `TradePlan` 版本或明确记录偏离。

## 页面原型

```text
┌──────────────────────────────────────────────────────────────────────┐
│ 交易计划 TP-20260723-001  待审阅  【仿真】  数据：10:31             │
├──────────────────────────────────────────────────────────────────────┤
│ 12 列决策条：为什么调整｜适用周期｜不行动后果｜失效条件             │
├───────────────────────────────────────┬──────────────────────────────┤
│ 7 列：候选动作与执行依赖              │ 5 列：组合影响与校验         │
│ 买入 A / 卖出 B / 保留 C              │ 当前 → 目标 → 计划后         │
│ 金额区间 / 顺序 / 状态                │ 风险 / 现金 / 费用 / 集中度  │
│ [排除] [改金额] [进交易台]            │ 授权 / 数据 / 风控           │
├───────────────────────────────────────┴──────────────────────────────┤
│ [暂不处理] [保存新版本] [生成订单草稿]                              │
└──────────────────────────────────────────────────────────────────────┘
```

布局约束：

- 顶部原因、周期、不行动后果和失效条件占满 12 列，不能压入窄侧栏；
- 主体使用 7/5，约 759/537 px；修改任一动作时，右侧立即进入“重新计算”；
- 重新计算完成前禁用“生成订单草稿”；
- 底部操作栏粘性显示，只有“生成订单草稿”是视觉主操作；
- 单标的计划可在 T03 使用 640–720 px 只读抽屉预览；进入编辑则使用本页。

## 页面功能

| 页面区域 | 功能模块 | 页面功能 | 主要呈现信息 | 用户操作 | AI处理 |
| --- | --- | --- | --- | --- | --- |
| 顶部栏 | 计划身份 | 展示计划唯一标识和状态 | 计划编号、版本、账户、仿真/实盘、创建者 | 复制链接、查看历史 | 读取，不修改 |
| 顶部栏 | 数据状态 | 展示输入是否仍有效 | 组合、授权、行情、研究版本和时点 | 刷新、查看过期项 | 判断需要重算的范围 |
| 原因区 | 调整目的 | 说明计划要解决的问题 | 目标、组合偏离、风险事件、用户请求 | 查看详情 | 用通俗语言总结 |
| 原因区 | 触发条件 | 展示计划产生原因 | 阈值、定期调仓、用户主动、策略更新 | 查看规则 | 解释，不改变规则 |
| 原因区 | 适用周期 | 展示建议持有与复核时间 | 开始、结束、复核日、失效条件 | 调整复核日 | 评估周期变化 |
| 原因区 | 不行动后果 | 对比保持现状 | 偏离、风险、现金、目标概率区间 | 展开假设 | 生成情景说明 |
| 组合区 | 配置对比 | 对比当前、目标和计划后组合 | 资产配置、地区、行业、币种、现金 | 切换维度 | 解释主要变化 |
| 组合区 | 风险影响 | 展示计划对风险的影响 | 风险预算、集中度、波动、回撤、压力情景 | 查看明细 | 生成解释，不替代计算 |
| 组合区 | 成本影响 | 展示费用和执行假设 | 手续费、滑点、税费占位、现金变化 | 调整金额 | 更新估算说明 |
| 动作区 | 候选动作 | 展示计划中的每个动作 | 标的、方向、目标变化、金额区间、顺序 | 排除、恢复、编辑 | 生成候选动作 |
| 动作区 | 单标的研究 | 查看每笔动作的依据 | 研究结论、反方观点、未知项、更新时间 | 进入 T03、查看 T10 | 载入或更新研究 |
| 动作区 | 执行顺序 | 说明先卖后买等依赖 | 依赖关系、现金约束、可并行性 | 调整允许范围 | 评估影响，规则服务校验 |
| 动作区 | 批次选择 | 选择本次执行范围 | 全部、部分、排除项、剩余偏离 | 勾选、取消 | 重新汇总计划 |
| 证据区 | 支持证据 | 展示支持计划的事实 | 来源、时点、适用范围 | 查看原始资料 | 总结事实 |
| 证据区 | 反方与未知 | 展示不确定性 | 反方观点、缺失数据、冲突、失效条件 | 反馈、请求更新 | 主动暴露不确定性 |
| 校验区 | 授权检查 | 检查计划是否在授权内 | 资产范围、风险预算、集中度、自主级别 | 查看、前往 T00 | 解释结果，不能覆盖 |
| 校验区 | 组合检查 | 检查目标权重和现金 | 权重和、现金下限、未解析资产 | 修正动作 | 说明阻断原因 |
| 校验区 | 数据检查 | 检查输入新鲜度 | 过期、缺失、来源冲突 | 刷新 | 更新受影响产物 |
| 历史区 | 版本比较 | 比较计划变更 | 字段差异、用户修改、AI 更新、时间 | 切换版本、恢复为新版本 | 总结差异 |
| 底部操作 | 延后/拒绝 | 不执行当前计划 | 原因、提醒时间、失效日期 | 延后、拒绝 | 记录反馈，停止当前执行 |
| 底部操作 | 保存新版本 | 保存用户修改 | 新版本号、变化摘要 | 保存 | 生成摘要，不确认 |
| 底部操作 | 生成草稿 | 生成单笔或批次订单草稿 | 草稿数量、账户、计划版本 | 生成、进入复核 | 只创建草稿 |

## 计划级别

| 类型 | 适用场景 | 推荐承载 |
| --- | --- | --- |
| 单标的手动计划 | 用户在 T03 主动制定交易策略 | 可在 T03 抽屉查看完整计划 |
| 单标的组合计划 | 组合偏离只涉及一个资产 | T04 或 T03 |
| 多标的调仓计划 | 组合再平衡、现金调整 | T04 独立页面 |
| 观察计划 | 当前不交易，只设置条件 | T04 摘要 + 提醒 |

## 关键状态

| 状态 | 页面表现 | 用户可执行操作 |
| --- | --- | --- |
| 草稿 | 标注尚未校验，不显示提交操作 | 编辑、放弃 |
| 待审阅 | 展示完整影响与证据 | 接受、修改、延后、拒绝 |
| 需修改 | 标出阻断项和受影响动作 | 修正、重新计算 |
| 已确认 | 计划字段只读，订单草稿独立 | 查看草稿、撤销未提交部分 |
| 执行中 | 展示批次进度和剩余偏离 | 进入 T07 |
| 部分完成 | 区分已完成和未执行动作 | 撤余、生成修正版 |
| 已完成 | 展示成交后组合 | 进入 T09 |
| 已失效 | 显示失效字段和时间 | 基于最新数据复制新计划 |
| 数据不足 | 不生成确定性动作 | 补数据、保存观察计划 |

## 开源项目补充

参考 TradingAgents、FinRobot 和 Hummingbot，计划增加结构化多空辩论、风险意见、组合裁决、配置版本和回测快照。回测只作为证据，研究 Agent 的批准也不等于用户交易确认。详细依据见 [开源项目参考与页面补充](02_开源项目参考与页面补充.md#55-t04-交易计划详情)。

## AI 与计划边界

| AI可以做 | AI不能做 |
| --- | --- |
| 生成计划草案和候选动作 | 确认计划或提交订单 |
| 解释组合变化与不行动后果 | 修改用户授权 |
| 暴露支持、反方证据和未知项 | 隐藏不利情景 |
| 根据用户修改重新解释 | 把风控阻断降级为提醒 |
| 生成版本差异摘要 | 将计划状态写成已执行 |

## 导航关系

- 入口：T01 待确认计划、T08 组合偏离、T03 单标的计划。
- 出口：单标的进入 T03；授权问题进入 T00；订单生成后进入 T03 复核层或 T07；完成后进入 T09。
- 返回后保留当前计划版本和折叠状态，但重新检查数据新鲜度。

## 验收标准

- 用户在看到订单参数前能找到调整目的、组合影响和不行动后果。
- 修改任一动作后，组合、费用、现金和风险结果同步重新计算。
- 硬风控阻断时不能生成可提交草稿。
- 已确认计划不可原地覆盖，修改必须生成新版本。
- 多标的计划能区分已完成、未执行和被用户排除的动作。
- 计划失效时保留历史，但不能继续使用旧校验提交。
