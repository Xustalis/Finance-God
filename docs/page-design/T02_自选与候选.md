# T02 自选与候选页面

## 页面定位

当前页面定位为 **用户主动关注资产与系统组合候选的统一研究入口**。标的范围对 PandaData
可用资产开放，界面必须展示资产类型、市场、实际数据频率和 Provider 时间。

“我的自选”表示用户主动关注；“组合候选”表示系统根据目标组合、授权和当前缺口提出的待研究资产。两者必须分开展示，候选不是买入指令。

## 与 AI 辅助交易台的关系

- 用户选择标的后进入 AI 辅助交易台，并携带来源、自选分组、候选原因和当前筛选条件。
- AI 辅助交易台完成标的研究与单标的交易操作。
- 本页不直接提交订单，不显示无法解释的单一推荐分数。

## 页面原型

```text
┌──────────────────────────────────────────────────────────────────────┐
│ 自选与候选  [搜索股票、ETF、基金]   [筛选]   数据：10:31            │
├──────────────────────────────────────────────┬───────────────────────┤
│ 8 列主表                                     │ 4 列持久 Inspector    │
│ [我的自选] [系统候选]                        │ 当前对象来源          │
│ 两个明确、互斥的列表状态                     │ 候选目的 / 组合作用   │
│ 名称 价格 持仓 风险 费用 流动性 数据质量     │ 风险 / 数据时点       │
│                                              │ [查看研究] [忽略]     │
├──────────────────────────────────────────────┴───────────────────────┤
│ 64 px 跨 Tab 比较托盘：最多 4 个标的                                │
└──────────────────────────────────────────────────────────────────────┘
```

布局约束：

- 1440 px 使用 8/4，约 869/427 px；
- “我的自选”和“系统候选”不得作为两张长表并排，使用带完整名称的互斥 Tab；
- Inspector 跨 Tab 保持，始终标明当前对象来自自选、系统候选或搜索；
- 选择 2–4 个标的后进入同路由全宽比较状态，不把比较表压入 4 列；
- 比较托盘高度 64 px，正文必须预留空间，不能遮挡最后一行。

## 页面功能

| 页面区域 | 功能模块 | 页面功能 | 主要呈现信息 | 用户操作 | AI处理 |
| --- | --- | --- | --- | --- | --- |
| 顶部栏 | 标的搜索 | 搜索 PandaData 支持的资产 | 代码、名称、市场、资产类型、支持状态、数据频率 | 搜索、选择 | 识别标的并加载已有研究 |
| 顶部栏 | 筛选器 | 按交易和组合条件筛选 | 市场、资产类型、费用、规模、流动性、风险、组合角色 | 设置、保存、清空 | 将自然语言转换为结构化条件 |
| 顶部栏 | 数据状态 | 展示候选数据是否可用 | 数据时点、来源、过期和冲突 | 刷新、查看来源 | 标记受影响字段 |
| 自选区 | 自选分组 | 管理主动关注资产 | 分组名称、数量、更新时间 | 新建、改名、排序、删除 | 可建议分组，不自动移动 |
| 自选区 | 自选列表 | 查看关注资产状态 | 价格、涨跌、持仓、风险、提醒、研究时间 | 排序、选择、移除 | 生成摘要和变化原因 |
| 自选区 | 自选备注 | 记录关注原因 | 用户备注、目标价、观察条件 | 编辑、搜索 | 可总结，不覆盖原文 |
| 自选区 | 研究状态 | 展示 AI 研究进度 | 未分析、分析中、已完成、过期、失败 | 请求分析、重试 | 执行结构化研究 |
| 自选区 | 提醒状态 | 管理价格和事件提醒 | 条件、有效期、通知渠道 | 新建、暂停、删除 | 可建议条件，不自动启用 |
| 候选区 | 候选目的 | 解释为什么出现候选 | 组合缺口、目标、授权、适用周期 | 查看原因 | 根据组合和授权生成候选 |
| 候选区 | 候选列表 | 展示待研究资产 | 组合适配、费用、规模、流动性、风险、数据质量 | 查看研究、比较、忽略 | 排序并解释 |
| 候选区 | 评分拆解 | 展示候选各维度 | 组合适配、风险、成本、流动性、证据完整度 | 展开、调整排序 | 计算可解释维度，不输出神秘总分 |
| 候选区 | 排除原因 | 说明不适合的标的 | 授权冲突、流动性不足、费用过高、数据不足 | 查看、反馈错误 | 解释排除，不放宽规则 |
| 候选区 | 忽略反馈 | 记录用户不感兴趣原因 | 暂不关注、已持有替代、观点不同、数据错误 | 选择、撤销忽略 | 调整后续候选排序 |
| 比较区 | 候选比较 | 对比最多 4 个资产 | 组合角色、费率、规模、跟踪误差、流动性、重叠、风险 | 添加、移除、进入研究 | 生成差异摘要 |
| 比较区 | 组合影响预览 | 预览替代或新增的影响 | 权重假设、集中度、风险、现金、费用 | 调整假设 | 生成预览，不创建计划 |
| 全局 | 加入自选 | 将候选转为用户关注 | 目标分组、来源、关注时间 | 选择分组、确认 | 触发候选研究任务 |
| 全局 | 进入交易台 | 打开单标的工作区 | 标的、来源、候选作用、当前账户 | 查看研究 | 将上下文传递给 T03 |

## 候选生成规则

| 维度 | 必须说明 | 不得作为单独依据 |
| --- | --- | --- |
| 组合适配 | 对哪个目标或偏离有帮助 | 单日涨幅 |
| 风险 | 波动、回撤、集中度和相关性影响 | “热门”标签 |
| 成本 | 费率、交易费用和潜在滑点 | 单一低费率 |
| 流动性 | 成交、规模和可交易性 | 历史收益 |
| 证据 | 数据来源、时点和完整度 | AI 主观置信度 |

## 关键状态

| 状态 | 页面表现 | 用户可执行操作 |
| --- | --- | --- |
| 空自选 | 提示搜索或从候选添加 | 搜索、查看候选 |
| 无符合候选 | 说明当前无需新增资产或筛选过严 | 调整筛选、查看组合 |
| 数据不足 | 候选卡显示缺失字段，不提供交易入口 | 查看来源、加入观察 |
| 来源冲突 | 并列冲突数据和影响 | 请求复核、暂时忽略 |
| 研究中 | 显示任务进度，不显示占位结论 | 离开、取消 |
| 研究失败 | 显示错误和 Trace ID | 重试、继续手动查看 |
| 标的不支持 | 显示市场或账户限制 | 查看原因 |
| 行情休市 | 保留研究和自选功能 | 设置提醒、进入研究 |

## 开源项目补充

参考 OpenBB 的数据 Provider 分层，每条候选事实增加来源、时点、市场时区、延迟等级和字段质量；已保存筛选器只保存条件，不复制候选事实。Provider 变化后旧研究必须进入待复核状态。详细依据见 [开源项目参考与页面补充](02_开源项目参考与页面补充.md#53-t02-自选与候选)。

## AI 与候选边界

| AI可以做 | AI不能做 |
| --- | --- |
| 根据目标、组合和授权生成候选 | 将候选标记为必买 |
| 解释候选和排除原因 | 仅因短期涨跌推荐 |
| 总结比较结果 | 隐藏反方证据和数据缺口 |
| 将自然语言筛选转为结构化条件 | 自动修改授权以容纳候选 |
| 根据用户反馈调整排序 | 自动加入自选或创建订单 |

## 导航关系

- 入口：T01 研究更新、T08 组合缺口、全局搜索。
- 出口：选择标的进入 T03；查看组合影响进入 T08；候选形成多标的调整时进入 T04。
- 从 T03 返回时恢复列表位置、筛选和比较栏。

## 验收标准

- 自选与系统候选具有明确标签，不混为一类。
- 每个候选都能说明与目标组合的关系、数据时点和主要风险。
- 候选数据不足或冲突时不能直接生成订单草稿。
- 搜索结果对同名跨市场标的显示代码、市场和资产类型。
- 忽略候选不会删除其证据和历史。
- 进入 T03 后能识别来源是自选、候选还是搜索。
