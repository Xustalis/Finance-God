# Finance-God 交易页面最终版设计（减法后版本）

> 状态：已减法优化。强制视觉、交互、文案、数据和验收规则见 `04_Finance-God前端设计要求.md` 和 `06_Finance-God页面减肥与前后端职责规范.md`。发生冲突时以这些规范为准。

## 1. 产品边界（减法后）

Finance-God 是面向普通投资者与专业交易用户的桌面交易产品。本版覆盖核心功能，额外能力已下沉或合并：

- T00 账户、授权与设置（简化）
- T01 总览与决策收件箱（简化）
- T02 行情、自选与候选（只保留搜索 + 基础行情）
- T03 AI 辅助交易台（简化为当前标的研究 + 草稿主操作）
- T04 调仓计划（简化）
- T07 订单与成交（简化为当前异常订单）
- T08 组合与偏离（简化）
- T09 交易复盘（简化）
- T10 数据目录（简化，只浏览）

不设计移动端。1440 px 是主要设计宽度，1024 px 是最低支持宽度。

## 2. 事实边界（减法后）

| 范围 | 事实来源 | UI 要求（减法后） |
| --- | --- | --- |
| 行情、分时、标的元数据 | PandaData | 只展示必需字段 + 来源、频率、时点、错误 |
| 仿真账户、持仓、订单 | Finance-God 仿真状态 | 持续显示“仿真”，仅显示当前对象必需事实 |
| AI 研究 | 页面上下文和研究产物 | 事实、推断、反方证据、未知项只在 Inspector 或 AI 侧栏显示 |
| 主要操作结果 | 后端命令 | 页面只展示结果和状态，不自行计算 |

## 3. 视觉与应用骨架（减法后）

视觉采用浅色、高密度、专业工作区：

- 白、灰、黑建立层级；蓝色只用于选择和主要操作。
- 主要结构使用边框、分隔线和紧凑表格，不堆叠卡片。
- 数字右对齐等宽；正文与关键交互不低于 13 px。
- 页面保持一个主纵向滚动区域。

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ Logo｜总览 行情 交易台 组合 订单 复盘 数据 设置｜仿真账户｜PandaData｜AI │
├──────────────────────────────────────────────────────────────────────────────┤
│ 页面标题｜当前对象 + 唯一主操作                                                         │
├───────────────┬───────────────────────────────────────┬──────────────────────┤
│ 左侧上下文     │ 主任务工作区（当前对象事实 + 图表）     │ 详情 / Inspector     │
│ 可隐藏、可调宽 │ 表格、图表、证据或计划                │ 可隐藏、可调宽       │
└───────────────┴───────────────────────────────────────┴──────────────────────┘
```

一级导航固定。全局不使用大型左侧导航。页面内部使用可排序工作区标签；所有页面共用常驻右侧 AI 侧栏，宽屏默认展开，1024–1279 px 默认收起为可见窄栏。用户选择保存到浏览器。

## 4. 页面布局与主操作（减法后）

| 页面 | 页面布局（减法后） | 主操作 |
| --- | --- | --- |
| T00-T02 | 标题 + 当前对象 + 单一主要按钮 | “在交易台打开”“提交计划”“查询行情” |
| T03 交易台 | 标的图表 + 研究/草稿 | 研究态打开草稿；草稿态进入复核 |
| T07 订单 | 订单表 + 核心提交按钮 | 处理当前异常订单 |
| T08 组合 | 资产摘要 + 持仓 + 调仓入口 | 创建调仓计划 |
| T09 复盘 | 计划、执行、结果和证据 | 完成本次复盘 |
| T10 数据 | PandaData 模块 / 接口目录 | 浏览为主，无强制主操作 |

T06 是隔离复核层：背景不可编辑，必须显示必需信息。

## 5. 工作区能力（减法后）

桌面工作区必须支持：

- 显示或隐藏左、中、右面板（但默认最小）
- 拖动分隔条改变面板宽度
- 调整工作区标签顺序
- 保存当前布局到浏览器
- 恢复默认布局

布局设置只有一个事实源。页面切换、刷新后不得出现两套面板状态。

## 6. 文案标准（减法后）

标题说明“用户在这里完成什么”，区域说明解释可见内容、主要操作和操作结果。禁止用抽象口号。

示例合格文案：
- “平安银行 000001.SZ 当前行情与 1 分钟图表”
- “提交调仓计划将生成仿真订单草稿”

## 7. 减法后版本特点

- 页面复杂度已下沉到后端。
- 前端只展示必需信息和唯一主操作。
- 所有非核心功能已合并或移除。
- 验收时页面必须能通过 `前端设计验收模板.md` 的所有通过项。
