# Finance-God 交易页面最终版设计

> 状态：当前有效。强制视觉、交互、文案、数据和验收规则见
> [Finance-God 前端设计要求](04_Finance-God前端设计要求.md)；发生冲突时以该规范为准。

## 1. 产品边界

Finance-God 是面向普通投资者与专业交易用户的桌面交易产品。本版覆盖：

- T00 账户、授权与设置；
- T01 总览与决策收件箱；
- T02 行情、自选与候选；
- T03 AI 辅助交易台，以及内部 T05 订单草稿、T06 最终复核；
- T04 调仓计划；
- T07 订单与成交；
- T08 组合与偏离；
- T09 交易复盘；
- T10 数据目录与过程证据。

不设计移动端。1440 px 是主要设计宽度，1024 px 是最低支持宽度；低于 1024 px 时显示桌面端访问提示。

## 2. 事实边界

| 范围 | 事实来源 | UI 要求 |
| --- | --- | --- |
| 行情、分时、标的元数据、数据目录 | PandaData | 展示来源、频率、Provider 时间、请求时间和错误 |
| 仿真账户、资金、持仓 | Finance-God 仿真状态 | 持续显示“仿真”，不得暗示真实账户 |
| 订单、成交、撤单、复盘 | Finance-God 仿真状态 | 明确区分提交中、状态未知、受理、部分成交和完成 |
| AI 研究与解释 | 页面上下文和研究产物 | 事实、推断、反方证据、未知项和失效条件分层 |

前端可每 1 秒检查行情，但页面展示的数据频率必须以 PandaData 实际返回为准。后端集中缓存并合并上游刷新；
禁止把重复读取同一快照描述成“1 秒级行情”，也禁止 API 失败后静默使用硬编码价格。

## 3. 视觉与应用骨架

视觉采用浅色、高密度、OKX 风格的专业工作区：

- 白、灰、黑建立层级；蓝色只用于选择和主操作；
- 红、绿只表达交易方向或风险状态，并辅以文字和图标；
- 主要结构使用边框、分隔线和紧凑表格，不堆叠营销式卡片；
- 数字右对齐并使用等宽数字；正文与关键交互不低于 13 px；
- 页面保持一个主纵向滚动区域，不制造三个互相竞争的纵向滚动轴。

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ Logo｜总览 行情 交易台 组合 订单 复盘 数据 设置｜仿真账户｜PandaData｜AI │
├──────────────────────────────────────────────────────────────────────────────┤
│ 页面标题｜对象与数据范围｜唯一主操作                                         │
├───────────────┬───────────────────────────────────────┬──────────────────────┤
│ 左侧上下文     │ 主任务工作区                          │ 详情 / 研究 / 草稿     │
│ 可隐藏、可调宽 │ 表格、图表、证据或计划                │ 可隐藏、可调宽         │
└───────────────┴───────────────────────────────────────┴──────────────────────┘
```

一级导航固定为：总览、行情、交易台、组合、订单、复盘、数据、设置。全局不使用大型左侧导航。
页面内部使用可排序工作区标签；所有页面共用常驻右侧 AI 侧栏，宽屏默认展开，
1024–1279 px 默认收起为可见窄栏。用户选择保存到浏览器；不使用遮罩抽屉或悬浮聊天按钮。

## 4. 页面布局与主操作

| 页面 | 页面布局 | 主操作 |
| --- | --- | --- |
| T00 设置 | 账户状态 / 授权边界 / 工作区设置 | 保存设置 |
| T01 总览 | 风险与组合摘要 / 决策收件箱 / 最近活动 | 处理最高优先级事项 |
| T02 行情 | 自选与全资产结果 / 标的详情 | 在交易台打开 |
| T03 交易台 | 标的与图表 / 研究或订单草稿 / 当前标的事实 | 研究态打开草稿；草稿态进入复核 |
| T04 计划 | 调仓原因 / 动作表 / 组合影响与校验 | 确认计划并生成草稿 |
| T07 订单 | 订单表 / 事件和详情 | 处理当前异常订单 |
| T08 组合 | 仿真资产摘要 / 持仓 / 偏离和计划入口 | 创建调仓计划 |
| T09 复盘 | 计划、执行、结果和证据阶梯 | 完成本次复盘 |
| T10 数据 | PandaData 模块 / 接口目录 / 接口说明 | 浏览为主，无强制主操作 |

T06 是隔离复核层：背景不可编辑，必须显示仿真账户、标的、方向、价格、数量、金额、
行情来源与时点。风控阻断时不显示提交按钮。

## 5. 工作区能力

桌面工作区必须支持：

1. 显示或隐藏左、中、右面板；
2. 拖动分隔条调整面板宽度；
3. 调整工作区标签顺序；
4. 保存当前布局到浏览器；
5. 恢复系统默认布局。

布局设置只有一个事实源。页面切换、刷新和保存后不得出现两套面板状态。
AI 侧栏独立于页面面板配置并始终存在，只允许展开或收起。

## 6. 文案标准

标题说明“用户在这里完成什么”，区域说明解释可见内容、主要操作和操作结果。禁止用抽象口号代替功能介绍。

例如：

- 不合格：`授权、计划、订单、成交和复盘共用一条事实流`
- 合格：`按时间展示授权变更、计划、订单、成交和复盘记录；选择记录可查看详情`

按钮使用动作 + 对象，如“在交易台打开”“进入订单复核”“查询订单状态”。不得使用缺少对象的
“处理”“查看”“确认”等泛化词。

## 7. 状态与安全交互

- 行情：加载中、当前、延迟、过期、缺失、失败；
- AI：排队中、分析中、数据不足、来源冲突、完成、过期、失败；
- 计划：草稿、待审阅、需修改、已确认、执行中、部分完成、完成、取消、失效；
- 订单：草稿、提交中、状态未知、已受理、部分成交、已成交、撤单中、已撤、已拒、过期。

状态未知时只允许查询状态，不显示“重新提交”。行情、授权、组合或校验版本变化后，旧结果必须显示过期。
颜色不是状态的唯一编码；抽屉和模态层支持 Escape 关闭、焦点约束和焦点恢复。

## 8. 实现与验收

可交互原型见 [prototype](../../prototype/README.md)，必须由后端服务启动以接入真实 PandaData 行情。
每次页面设计或实现任务完成后，Agent 必须按
[前端设计验收模板](templates/前端设计验收模板.md) 逐条输出“通过 / 不通过 / 不适用”、
证据和遗留问题；未输出逐项结果不得声称完成。
