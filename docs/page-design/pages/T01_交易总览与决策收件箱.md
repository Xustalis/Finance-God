# T01 交易总览与决策收件箱

> 文档类型：页面规格 / Reference  
> 对应导航：总览  
> 适用宽度：1440 px 主画布，1024 px 最低宽度

## 页面任务与主对象

当前页面定位为 **交易闭环的状态总览和决策待办入口**。

用户应在 30 秒内知道当前账户是否安全、组合是否偏离、有哪些计划等待确认、订单是否异常，以及下一步需要处理什么。页面不以行情热榜或频繁交易为目标。

## 跨页关系

- 从候选、研究更新和单标的提醒进入 AI 辅助交易台。
- 汇总交易台产生的草稿、订单和当前标的异常，但不复制订单事实。
- 全账户订单状态来自 T07；组合偏离来自 T08；证据来自 T10。

## 信息层级与工作区

```text
┌──────────────────────────────────────────────────────────────────────┐
│ 总览  【仿真账户】 市场：交易中  数据：10:31:08  [暂停 Agent]        │
├───────────────────────────────────────┬──────────────────────────────┤
│ 7 列：组合、风险与数据                │ 5 列：决策收件箱             │
│ 总资产 / 现金 / 风险预算              │ P0 阻断                      │
│ 当前配置 vs 目标配置 / 偏离           │ P1 待确认计划与异常订单       │
│ 数据完整度 / 相关研究更新             │ P2 需关注 / P3 信息           │
├──────────────────────────────┴───────────────────────────────────────┤
│ 最近活动：计划、确认、订单、成交、撤单、复盘                        │
└──────────────────────────────────────────────────────────────────────┘
```

布局约束：

- 1440 px 使用 7/5，约 759/537 px；底部活动占满 12 列；
- 硬阻断存在时跨两列置顶，其视觉层级高于收益和市场变化；
- 首屏必须出现 P0/P1 事项；没有待办时明确显示“当前无需操作”；
- 每条待办只有一个下一步，不在总览内完成复杂下游任务。

## 功能与数据

| 页面区域 | 功能模块 | 页面功能 | 主要呈现信息 | 用户操作 | AI处理 |
| --- | --- | --- | --- | --- | --- |
| 顶部栏 | 账户模式 | 展示当前交易环境 | 仿真/实盘、账户、币种 | 切换账户 | 切换分析上下文 |
| 顶部栏 | 市场与数据 | 展示可交易性和数据新鲜度 | 市场状态、交易所时间、行情延迟、组合时点 | 查看说明、刷新 | 判断结论是否仍适用 |
| 顶部栏 | Agent 状态 | 展示自动化是否运行 | 正常、暂停、阻断、任务失败 | 暂停、查看任务 | 停止或恢复新任务 |
| 组合区 | 资产摘要 | 展示当前账户概况 | 总资产、现金、持仓、当期收益 | 切换周期 | 总结变化原因 |
| 组合区 | 目标与偏离 | 展示当前和目标差异 | 资产配置、风险预算、偏离值、目标状态 | 查看组合、查看计划 | 解释主要偏离 |
| 收件箱 | 待确认计划 | 汇总需要用户决策的计划 | 计划目的、影响、创建时间、到期时间 | 查看、延后、拒绝 | 排序并总结，不代确认 |
| 收件箱 | 订单待办 | 汇总订单异常和需处理状态 | 部分成交、撤单失败、拒单、状态未知 | 进入执行中心 | 解释状态，不修改事实 |
| 收件箱 | 授权待办 | 展示授权和账户问题 | 即将到期、需重新授权、账户断连 | 进入 T00 | 解释影响 |
| 收件箱 | 复盘待办 | 提醒需要完成的复盘 | 计划周期、执行完成时间、复盘原因 | 开始复盘、延后 | 生成复盘提纲 |
| 风险区 | 风险事件 | 展示影响交易的风险 | 集中度、现金、数据、市场和策略事件 | 查看详情 | 用通俗语言解释 |
| 风险区 | 数据质量 | 展示数据缺失与冲突 | 过期来源、未解析资产、持仓差异 | 修复、查看来源 | 标注受影响结论 |
| 市场区 | 相关事件 | 仅展示与组合和自选相关的信息 | 标的、事件、影响范围、时点 | 查看研究、忽略 | 归纳影响，不制造紧迫感 |
| 市场区 | 研究更新 | 展示新研究和结论变化 | 原结论、新结论、变化原因 | 进入交易台、查看证据 | 生成变化摘要 |
| 活动区 | 最近活动 | 汇总交易闭环事实 | 用户修改、计划、订单、成交、风控、复盘 | 筛选、查看详情 | 总结，不生成交易记录 |
| 全局 | 待办排序 | 根据严重度和时效排序 | 阻断、需确认、信息、已完成 | 调整偏好 | 依据规则排序，AI只做摘要 |
| 全局 | 已读与归档 | 管理已处理事项 | 已读、已处理、失效、归档时间 | 标记已读、归档 | 不删除审计记录 |

## 决策收件箱优先级

| 优先级 | 条件 | 示例 |
| --- | --- | --- |
| P0 阻断 | 资金、授权、账户或数据导致交易不可用 | 授权失效、账户断连、订单状态未知 |
| P1 需处理 | 用户需要确认或处置 | 调仓计划、部分成交、撤单失败 |
| P2 需关注 | 可能影响组合但无需立即操作 | 风险接近上限、研究结论变化 |
| P3 信息 | 已完成或一般提醒 | 成交完成、复盘报告生成 |

## 关键状态

| 状态 | 页面表现 | 用户可执行操作 |
| --- | --- | --- |
| 新用户/空组合 | 显示建档、授权和导入持仓主线 | 前往建档、T00、T08 |
| 正常且无待办 | 显示“当前无需操作”和下一复核时间 | 查看组合或研究 |
| 存在硬阻断 | 顶部持续状态条，相关入口优先 | 查看并修复 |
| 行情休市 | 显示下一交易时段，不制造倒计时 | 研究、保存计划 |
| 数据过期 | 标记受影响待办，不展示新确定性结论 | 刷新、查看来源 |
| Agent 失败 | 展示任务失败和 Trace ID | 重试、继续手动查看 |
| 账户切换中 | 骨架加载并清空旧账户操作入口 | 等待、取消切换 |
| 离线 | 展示最后同步时间，只读历史 | 重连 |

## AI 边界

| AI可以做 | AI不能做 |
| --- | --- |
| 汇总组合、计划和风险变化 | 创造不存在的待办 |
| 对待办信息进行解释和聚类 | 修改订单、持仓或授权状态 |
| 按用户任务和影响范围生成摘要 | 用涨跌制造交易紧迫感 |
| 说明为什么需要关注 | 把市场新闻直接变成下单指令 |
| 生成复盘提纲 | 自动标记用户已确认 |

## 导航关系

- T01 是交易闭环默认入口。
- 点击计划进入 T04；候选和研究进入 T02/T03；订单异常进入 T07；组合偏离进入 T08；复盘待办进入 T09；授权问题进入 T00。
- 返回 T01 后保持账户和筛选，但重新加载事实状态。

## 验收标准

- 用户能在 30 秒内找到最严重的阻断、待确认计划和异常订单。
- 待办数量与源对象一致，不因 AI 汇总重复创建待办。
- 无待办时明确显示“无需操作”，不填充无关机会。
- 任一风险或研究待办显示数据时点和影响范围。
- 已归档待办仍能从活动和审计中检索。
- 账户切换后不残留上一账户的金额、订单或主操作。
