# T03 AI 辅助交易台

> 文档类型：页面规格 / Reference  
> 对应导航：交易台  
> 适用宽度：1440 px 主画布，1024 px 最低宽度

## 页面任务与主对象

当前页面定位为 **AI 上下文感知的全资产仿真交易台**。标的通过 PandaData 全部可用数据集开放；
当前原型已验证 A 股、ETF，并按 Provider 能力支持基金、港股和美股。行情是真实数据，
账户、持仓、订单和成交是仿真事实。

AI 接收页面中的有效操作，用于当前标的分析、策略解释和风险检查，但最终交易由用户在 T06 确认。候选推荐和自选管理归 T02，本页不重复提供。

## 跨页关系

- 从 T02 接收当前标的、来源和筛选上下文；
- 从 T04 接收不可变计划版本；
- 将已确认的仿真订单交给 T07 跟踪；
- 组合影响进入 T08，研究和运行证据进入 T10；
- T05 订单草稿和 T06 最终复核是本页内部工作态，不创建一级路由。

## 信息层级与工作区

页面采用 **顶部一级导航 + 可配置三面板工作区 + 互斥工作态**。面板可隐藏、调整宽度，
工作区标签可排序并保存到浏览器。强制布局依据见
[前端统一设计规范](../01_前端统一设计规范.md)。

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ Logo｜总览 行情 交易台 组合 订单 复盘 数据 设置｜仿真｜PandaData｜AI     │
├──────────────────┬────────────────────────────────────┬──────────────────────┤
│ 标的与行情摘要    │ PandaData 分时图和交易事实          │ 研究判断 / 订单草稿    │
│ 可隐藏、可调宽    │ 当前标的持仓、委托与成交            │ 两个互斥工作态         │
└──────────────────┴────────────────────────────────────┴──────────────────────┘
└──────────────────────────────────────────────────────────────────────────────┘
```

### 1440 px 规格

- 顶部使用 8 个文字一级导航，不使用大型左侧全局导航；
- 页面使用左、中、右三面板；面板可隐藏和调宽；
- 左栏显示当前标的与行情事实，中栏显示图表和交易事实，右栏显示研究或订单任务；
- 研究态唯一主操作为“创建/打开订单草稿”；
- 订单态唯一主操作为“进入最终复核”；
- AI 深度解释使用全部页面共用的常驻右侧栏；侧栏可收起但入口始终可见；
- 最终复核使用最大宽 840 px 的 T06 覆盖层，背景不可编辑；硬阻断时不显示提交按钮。

### 计划锚点必须呈现

| 字段 | 作用 |
| --- | --- |
| 结构化结论 | 说明当前是买入、卖出、持有、等待或数据不足 |
| 适用周期 | 防止不同周期的策略与订单混用 |
| `TradePlan` 版本 | 明确草稿依据 |
| 研究与行情时点 | 识别过期或不同步 |
| 草稿相对计划的偏离 | 修改参数后持续显示影响 |
| 过期、冲突和阻断 | 抑制不合法主操作 |

### 桌面宽度

- 1440 px：完整三面板工作区。
- 1024 px：保留三面板任务，缩短非关键状态文字；不得隐藏模式、账户、数据时点、硬阻断和订单状态。
- 低于 1024 px：显示桌面端访问提示，不提供移动版。

## 功能与数据

| 页面区域 | 功能模块 | 页面功能 | 主要呈现信息 | 用户操作 | AI处理 |
|---|---|---|---|---|---|
| 顶部栏 | 标的搜索 | 搜索 PandaData 支持的资产 | 代码、名称、市场、资产类型、数据频率 | 搜索、选择、切换标的 | 接收当前标的，加载或生成分析 |
| 顶部栏 | 行情摘要 | 展示当前交易行情 | 最新价、涨跌额、涨跌幅、买一、卖一、价差、今开、最高、最低、昨收、成交量 | 刷新行情 | 将最新行情纳入分析上下文 |
| 顶部栏 | 市场状态 | 显示当前是否可交易 | 开市、休市、盘前、盘后、交易所时间、行情延迟 | 查看状态说明 | 判断策略和订单是否适用于当前时段 |
| 顶部栏 | 账户摘要 | 展示交易账户状态 | 模拟/实盘、账户、可用资金、购买力、当前持仓、浮动盈亏 | 切换账户 | 根据账户与持仓调整风险判断 |
| K线区 | K线行情 | 查看价格走势 | 蜡烛图、成交量、实时价格线 | 缩放、拖动、十字光标、回到最新 | 接收当前可见周期和价格区间 |
| K线区 | 周期切换 | 切换K线粒度 | 1分、5分、15分、1小时、日、周 | 选择周期 | 检查当前策略是否适用于新周期 |
| K线区 | 时间范围 | 调整历史行情范围 | 1日、5日、1月、6月、年初至今、1年 | 选择范围 | 更新趋势与波动分析上下文 |
| K线区 | 技术指标 | 查看技术分析指标 | MA、EMA、BOLL、MACD、RSI | 添加、隐藏、调整参数 | 将用户选择的指标纳入分析 |
| K线区 | AI策略图层 | 可视化AI生成的策略 | 入场区、止损线、目标线、失效线 | 显示、隐藏、点击查看说明 | 将当前策略映射到K线 |
| K线区 | 交易图层 | 展示仿真交易信息 | 持仓成本线、未成交委托线、买入点、卖出点 | 点击查看订单或成交详情 | 使用订单与持仓更新策略上下文 |
| 下单区 | 交易方向 | 选择交易动作 | 买入、卖出 | 切换买卖方向 | 检查是否与当前策略一致 |
| 下单区 | 订单类型 | 设置成交方式 | 市价单、限价单 | 切换订单类型 | 评估价格偏离和成交风险 |
| 下单区 | 数量输入 | 设置交易规模 | 按股数/金额、数量、限价、可买/可卖数量 | 输入数值，使用25%/50%/75%/最大比例 | 计算仓位和账户风险 |
| 下单区 | 有效期 | 设置订单有效时间 | 当日有效、撤销前有效 | 选择有效期 | 检查是否符合策略适用周期 |
| 下单区 | 风险参数 | 设置交易风险边界 | 止损价、止盈价、最大亏损、风险收益比 | 手动填写或采用AI参数 | 实时检查参数和策略偏离 |
| 下单区 | 订单估算 | 预估订单结果 | 交易金额、手续费、总支出/到账、剩余资金、成交后持仓 | 查看估算结果 | 检查余额、仓位和集中度 |
| 下单区 | AI填入草稿 | 使用AI策略参数生成订单草稿 | AI入场价、数量、止损、止盈 | 填入草稿、修改参数 | 只填充订单，不自动提交 |
| 下单区 | 进入最终复核 | 将不可编辑的最终摘要交给 T06 | 完整订单摘要、风险提示、当前行情 | 进入 T06 | 不提交订单；T06 确认后才创建 `OrderIntent` |
| 底部区 | 当前持仓 | 查看已有仓位 | 数量、可卖数量、平均成本、当前价、市值、今日盈亏、累计盈亏、仓位占比 | 选择持仓、快速填充卖出订单 | 分析集中度和持仓风险 |
| 底部区 | 当前委托 | 管理未完成订单 | 方向、类型、价格、数量、已成交、剩余数量、状态、时间 | 修改、撤单、定位到K线 | 根据订单状态更新策略 |
| 底部区 | 成交记录 | 查看实际成交 | 时间、方向、均价、数量、金额、手续费、订单编号 | 查看详情、定位到K线 | 更新持仓上下文，不生成AI交易记录 |
| AI侧栏 | 当前标的分析 | 分析正在查看的资产 | 综合观点、技术面、基本面、市场环境、用户适配度 | 展开依据、请求更新、向AI追问 | 根据当前页面上下文生成分析 |
| AI侧栏 | 当前策略 | 制定当前标的交易策略 | 买入/卖出/等待、策略类型、入场条件、止损、目标、仓位上限、适用周期 | 采用、调整、重新生成 | 生成结构化交易策略 |
| AI侧栏 | 分析依据 | 解释AI结论 | 支持理由、反方观点、主要风险、未知项、失效条件 | 查看证据、追问原因 | 将事实、推断和风险分开呈现 |
| AI侧栏 | 策略操作 | 将AI策略应用到交易区 | 策略参数与当前订单差异 | 显示到K线、填入订单草稿 | 传递参数，不执行交易 |
| AI侧栏 | AI对话 | 根据当前交易上下文继续交流 | 当前标的、周期、指标、策略、持仓和订单上下文 | 输入筛选、分析、调参和解释请求 | 持续理解页面中的有效操作 |
| 全局 | 操作感知 | 将有效用户行为同步给AI | 标的、周期、指标、自选、订单草稿、委托和持仓变化 | 正常操作页面 | 接收结构化事件，不采集无意义鼠标行为 |
| 全局 | 策略偏离提醒 | 检查订单是否偏离AI策略 | 原策略参数、当前订单参数、风险差异 | 保持修改、恢复参数、重新计算 | 解释修改带来的风险变化 |
| 全局 | 交易校验 | 防止无效订单 | 休市、行情过期、资金不足、数量不足、价格偏离、标的不支持 | 修正或取消订单 | 补充风险说明，但不阻止合法手动交易 |
| 全局 | 状态反馈 | 显示系统和订单状态 | AI分析中、数据不足、订单提交中、部分成交、已成交、撤销、拒单 | 重试、撤单或继续手动操作 | AI不可用时，交易功能仍保持可用 |

## 关键状态

| 状态 | 页面表现 | 用户可执行操作 |
| --- | --- | --- |
| 行情可用 | 显示 PandaData 来源、时点、频率和新鲜度 | 研究、编辑草稿 |
| 行情过期 | 保留最后成功值并明确标记过期 | 刷新、只读研究 |
| 行情失败 | 显示错误和影响范围，不回填演示价格 | 重试、查看历史 |
| 无仿真账户 | 禁用订单工作态并说明原因 | 进入 T00 创建账户 |
| 研究失败 | 手动草稿能力保持可用，AI 区显示失败 | 重试研究、手动编辑 |
| 草稿过期 | 保留输入但禁止进入最终复核 | 重新计算 |
| 硬风控阻断 | 显示规则、版本和恢复条件 | 修改草稿、放弃 |
| 提交中 | 锁定重复提交入口 | 等待、查看状态 |
| 状态未知 | 禁止重新提交 | 进入 T07 对账 |

## AI 边界

| AI可以做 | AI不能做 |
|---|---|
| 接收页面中的有效操作上下文 | 自动提交订单 |
| 分析从 T02 进入的 PandaData 支持标的 | 未经确认修改已提交订单 |
| 解释当前标的与组合的关系 | 自动买入或卖出 |
| 制定入场、止损、目标和仓位策略 | 在后台自动交易 |
| 将策略显示到K线 | 将策略信号伪装成成交 |
| 将策略参数填入订单草稿 | 创建“AI交易记录” |
| 检查订单风险和策略偏离 | AI 故障时绕过数据、验证、风控或 T06 确认 |

## 导航关系

- 入口：T02 当前标的、T04 单标的计划、T08 持仓、T01 研究待办。
- 出口：订单进入 T07；组合影响进入 T08；证据进入 T10；授权问题进入 T00。
- 返回时保留标的、图表范围、工作态和未提交草稿，但重新加载行情、账户与能力状态。

## 验收标准

- 行情明确标注 PandaData 来源、供应商时点、实际频率和请求间隔。
- 真实行情与仿真账户、持仓、订单和成交保持独立状态与明确标签。
- 行情刷新不重置标的选择、图表范围或订单输入。
- 研究态和订单态互斥，页面始终只有一个视觉主操作。
- 订单只能通过 T06 最终复核提交；状态未知时不能重复提交。
- AI 可填充草稿但不能修改业务事实、绕过风控或自动提交订单。
