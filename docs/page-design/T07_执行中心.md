# T07 执行中心页面

## 页面定位

当前页面定位为 **全账户订单、成交、撤单、拒单和批次执行的事实中心**。

页面只展示来自仿真撮合或经纪商回报的执行事实。AI 可以解释状态和影响，但不能创建成交记录、推断订单已受理或静默重试失败订单。

## 与 AI 辅助交易台的关系

- AI 辅助交易台底部只展示当前标的的持仓、委托和成交摘要。
- 本页展示全账户、跨标的和多批次执行状态。
- 用户从交易台提交订单后进入本页跟踪；从本页可返回交易台修改并创建新草稿。

## 页面原型

```text
┌──────────────────────────────────────────────────────────────────────┐
│ 执行中心 【仿真账户】 [全部] [处理中] [部分成交] [异常]  10:31     │
├──────────────────────────────────────────────────────────────────────┤
│ 批次 RB-001  2/3 笔完成  计划 TP-001  剩余组合偏离 1.1%            │
├───────────────────────────────────────┬──────────────────────────────┤
│ 默认 12 列；选中后 8 列订单主表       │ 4 列只读 Inspector          │
│ 时间 标的 方向 价格/数量 成交 状态    │ [时间线][成交][风控][影响]  │
│ 10:32 510300 买入 x/1000 600 部分成交 │ 订单、外部编号、可撤数量    │
│ 10:30 xxxxxx 卖出 x/500  500 已成交   │ [撤单/撤余]                 │
└──────────────────────────────────────────────────────────────────────┘
```

布局约束：

- 未选中订单时主表占满 12 列；选中后切换为 8/4，不跳转路由；
- 批次进度和最近执行回报时点固定在表格上方；
- Inspector 只展示选中订单，取消选择后恢复全宽表；
- 撤单和撤余使用短确认层；状态未知时 Inspector 不提供重新提交入口；
- Inspector 中的 AI 解释必须与执行事实分层，不能伪装成订单状态。

## 页面功能

| 页面区域 | 功能模块 | 页面功能 | 主要呈现信息 | 用户操作 | AI处理 |
| --- | --- | --- | --- | --- | --- |
| 顶部栏 | 账户选择 | 切换执行账户 | 仿真/实盘、账户、币种、连接状态 | 切换 | 切换解释上下文 |
| 顶部栏 | 市场状态 | 展示订单所处交易时段 | 市场、交易所时间、开休市、延迟 | 查看说明 | 解释时段影响 |
| 顶部栏 | 状态筛选 | 筛选订单和批次 | 处理中、部分成交、完成、异常、历史 | 选择、保存筛选 | 不改变事实 |
| 顶部栏 | 搜索与导出 | 检索执行记录 | 标的、订单号、批次、计划、外部编号 | 搜索、导出 | 可生成摘要 |
| 摘要区 | 执行概览 | 汇总当前订单状态 | 待受理、未成交、部分成交、异常、当日成交 | 点击筛选 | 解释主要变化 |
| 批次区 | 批次进度 | 展示多笔调仓执行 | 完成数、失败数、剩余动作、现金、偏离 | 展开、进入计划 | 总结批次状态 |
| 批次区 | 依赖状态 | 展示订单顺序和阻塞 | 等待前序成交、现金不足、市场未开 | 查看原因 | 解释依赖，不跳过 |
| 订单表 | 订单列表 | 查看全账户订单 | 时间、标的、方向、类型、价格、数量、成交、状态 | 排序、选择 | 不生成订单事实 |
| 订单表 | 快捷操作 | 对合法状态执行操作 | 可撤数量、可修改字段、市场限制 | 撤单、撤余、查看 | 说明后果，不自动操作 |
| 详情区 | 订单摘要 | 查看订单完整参数 | 账户、方向、类型、数量、价格、有效期、来源 | 复制编号、查看计划 | 解释与计划差异 |
| 详情区 | 事件时间线 | 查看状态转换 | 草稿、确认、提交、受理、成交、撤单、拒单时间 | 展开事件 | 总结，不补造事件 |
| 详情区 | 成交明细 | 查看每次实际成交 | 时间、价格、数量、费用、流动性、回报编号 | 导出、定位图表 | 计算执行摘要 |
| 详情区 | 校验记录 | 查看提交前规则结果 | 授权、资金、价格、市场、数据版本 | 查看 T10 | 解释命中原因 |
| 详情区 | 拒单信息 | 展示失败原因 | 来源、错误码、原始信息、可修正字段 | 创建修正版草稿 | 将错误转为可理解说明 |
| 详情区 | 撤单状态 | 跟踪撤单请求 | 请求时间、外部状态、剩余数量 | 刷新、查看 | 不提前显示已撤 |
| 详情区 | 组合影响 | 展示实际成交后的变化 | 现金、持仓、权重、剩余偏离 | 查看 T08 | 总结实际影响 |
| 全局 | 状态刷新 | 获取最新执行事实 | 最近回报时间、连接状态 | 手动刷新 | 不缓存为新事实 |
| 全局 | 异常恢复 | 处理状态未知和连接异常 | 查询次数、最后状态、支持信息 | 重新查询、联系支持 | 解释，不重复提交 |

## 订单状态

| 状态 | 定义 | 可用操作 |
| --- | --- | --- |
| 提交中 | 请求正在发送，尚无受理结果 | 等待，禁止重复提交 |
| 状态未知 | 请求结果不确定 | 查询状态，禁止重下 |
| 已受理 | 执行方已接受，尚未完全成交 | 查看、符合条件时撤单 |
| 部分成交 | 仅部分数量成交 | 撤余、继续等待 |
| 已成交 | 全部成交 | 查看成交与复盘 |
| 撤单中 | 撤单请求已发送 | 等待，不显示已撤 |
| 已撤 | 剩余数量已确认撤销 | 查看、按需创建新草稿 |
| 已拒 | 执行方拒绝订单 | 查看原因、创建修正版 |
| 已过期 | 有效期结束未完全成交 | 查看、创建新草稿 |

## 关键状态

| 场景 | 页面表现 | 用户可执行操作 |
| --- | --- | --- |
| 无订单 | 显示“暂无执行记录”及 T03/T04 入口 | 前往研究或计划 |
| 连接正常 | 显示最近回报时间 | 正常操作 |
| 连接中断 | 顶部持续警告，状态标记可能过期 | 重新连接、只读历史 |
| 批次部分完成 | 区分完成、等待和失败动作 | 查看剩余偏离 |
| 订单状态未知 | 高优先级阻断重复操作 | 主动查询、联系支持 |
| 撤单失败 | 保留原订单状态并展示原因 | 重试合法请求、继续等待 |
| 回报冲突 | 并列原始回报并暂停组合更新 | 请求人工复核 |
| 仿真模型异常 | 明确停止撮合，不伪造成交 | 查看模型错误、稍后重试 |

## 开源项目补充

参考 FreqUI 和 Hummingbot，执行中心增加连接健康条、实时事件状态、订单状态时间线、脱敏错误与 Trace ID，以及“暂停创建新订单”。事件流中断必须进入“状态未知”，恢复后先对账，不能静默重试。详细依据见 [开源项目参考与页面补充](02_开源项目参考与页面补充.md#56-t07-执行中心)。

## AI 与执行边界

| AI可以做 | AI不能做 |
| --- | --- |
| 解释订单状态、拒单和费用 | 创建或修改订单事实 |
| 总结批次进度和剩余偏离 | 将提交中显示为已受理 |
| 根据拒单原因生成修正版草稿建议 | 静默重试或重复提交 |
| 解释成交与计划差异 | 生成“AI 成交记录” |
| 帮助检索历史订单 | 将撤单中显示为已撤 |

## 导航关系

- 入口：T01 订单待办、T03 提交结果、T04 批次执行。
- 出口：标的进入 T03；计划进入 T04；组合影响进入 T08；完成批次进入 T09；证据进入 T10。
- 返回后保留账户、筛选和选中订单。

## 验收标准

- “请求已发送”“已受理”“已成交”是三个独立状态。
- 重复点击、页面刷新和网络重试不会创建重复订单。
- 部分成交准确显示成交数量、剩余数量和均价。
- 撤单中不能被展示为已撤。
- 状态未知时禁止用户通过同一草稿重复下单。
- 任一订单都能关联计划、草稿、确认、校验、成交和外部回报编号。
