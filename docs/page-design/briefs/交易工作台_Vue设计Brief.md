# 交易工作台 Vue 设计 Brief

## 用户与操作环境

- 面向能够独立判断的零售用户与专业交易者，核心环境是桌面浏览器。
- 1440 px 为主画布，1024 px 为最小工作宽度；低于 1024 px 只显示明确的桌面宽度提示，不提供移动布局。
- 工作台强调高信息密度、清晰状态和可追溯操作，不使用营销式首屏、卡片墙或装饰性大标题。

## 页面任务与主对象

- 全局任务：在一个统一产品壳中完成行情查看、标的研究、订单草稿、组合检查、执行跟踪、复盘与证据追溯。
- T01 主对象是唯一关联对象的待办；T02 是自选标的；T03 是当前标的与交易计划锚点；T04 是不可变计划版本；T07 是订单/批次；T08 是组合版本；T09 是复盘记录；T10 是对象版本与证据节点。
- T05 订单草稿是 T03 的互斥工作态，T06 最终复核是最大宽 840 px 的不可编辑覆盖层，不新增独立路由。
- 所有桌面路由共享同一个 AI 上下文侧栏；侧栏只解释当前页面和选中对象，不修改业务事实、不提交订单。

## 信息层级与工作区

1. 48 px 全局顶栏：产品标识、八个一级路由、仿真模式、PandaData 与 AI 状态。
2. 44 px 页面工具栏：页面标题、当前对象、上游时点、数据频率、刷新节奏和页面唯一主操作。
3. 可选 36 px 工作区标签：同一路由内的互斥任务态，支持排序与浏览器持久化。
4. 主工作区：使用列表、图表、语义表格、分隔线和持久检查器；相关面板支持显隐、鼠标调整、键盘调整与重置。
5. 独立 AI 列：宽度 288–420 px，可调整；收起后保留 44 px 可见轨道。1440 宽默认展开，1024–1279 宽默认收起，用户选择跨路由和刷新保留。

## 数据、频率、新鲜度与失败状态

- 行情唯一前端事实源是后端规范化 PandaData API：`/api/market/quotes`、`/api/market/bars`、`/api/market/catalog`。
- P0 图表只呈现后端返回的实际频率；当前后端日线为主，不显示未接入的分钟或实时能力。
- 行情区域持续展示 `provider_time`、实际 `frequency`、`freshness`、市场发布状态和前端请求间隔。
- 使用一个共享轮询控制器与缓存：同一请求去重；页面隐藏时停止；重新可见时立即刷新；保留最后一次成功值并明确标记 stale/error。
- 后端错误、空结果或不完整响应显示“不可用/数据异常”，不得回填演示价格、最近值或静态成功状态。
- 账户、持仓、订单、成交和执行数据与行情状态分开存储，并始终标记“仿真”。当前交易 API 未完成时，相关页面稳定复现“交易服务不可用”，不伪造订单、持仓或复盘结果。

## 主操作与启用条件

- 总览：进入最高优先级且具有唯一对象引用的待办；无真实对象时禁用。
- 行情：在交易台打开当前具有可用 PandaData 行情的标的。
- 交易台研究态：创建/打开订单草稿；需要当前标的与可用行情，AI 失败不阻止手动草稿。
- 交易台订单态：进入最终复核；需要草稿字段完整、行情未过期且不存在硬阻断。
- 最终复核：只有正式风控通过且交易 API 可用时显示提交操作；本切片交易 API 未接通，因此不显示提交按钮。
- 计划、组合、执行与复盘：分别以确认计划并生成草稿、生成交易计划、跟踪订单、保存复盘为主操作；缺少服务事实时禁用并显示原因。
- 数据与证据：以查看来源/对象版本为主；P0 不提供导出。

## 桌面宽度行为

- 1440 px：完整主工作区与默认展开 AI 列；交易台显示三面板任务区。
- 1280–1439 px：保留完整任务信息，AI 仍默认展开并允许用户收起。
- 1024–1279 px：保留所有关键状态与操作，AI 首次访问默认显示 44 px 轨道，主工作区压缩非关键列。
- 低于 1024 px：覆盖显示桌面访问提示，禁止误用压缩后的交易操作界面。
