# 心智驱动 AI 投资顾问 — 产品设计方案

## 一、产品定位与价值主张

### 1.1 产品定位
**"懂你心理的 AI 投资伙伴"** — 通过实时感知用户心理状态，结合多市场情绪洞察，提供个性化且动态调整的投资策略。

### 1.2 核心价值主张
- **心智感知**：不是问你想要什么，而是感知你真正能承受什么
- **动态适配**：策略随你的心理状态和市场情绪实时调整
- **多市场覆盖**：大A、美股、港股的统一策略视角
- **认知增强**：帮你识别自己的认知偏差，做出更理性的决策

### 1.3 差异化定位
| vs 传统投顾 | vs Betterment等 | vs 量化交易工具 |
|------------|----------------|---------------|
| 7×24在线，无情绪偏差 | 真正的实时心理感知 | 关注人而非纯市场 |
| 成本低10倍 | 动态策略调整 | 长期投资导向 |
| 数据驱动 | 多市场覆盖 | 可解释性强 |

### 2.2 Agent 间协作机制

```
协作流程示例（市场大跌场景）：

Agent2(市场情绪) ──检测到恐慌指数飙升──→ Orchestrator
                                              │
Agent1(用户心智) ──检测到用户焦虑上升──→ Orchestrator
                                              │
Orchestrator 综合判断：                        │
  • 市场：恐慌但非系统性风险                   │
  • 用户：焦虑度高，有损失厌恶倾向             │
  • 决策：触发"安抚+保守调整"策略              │
                                              ▼
Agent3(投资策略) ←── 接收指令：
  • 降低风险敞口5-10%
  • 增加现金/债券比例
  • 生成用户可理解的调整理由
                    │
                    ▼
              输出给用户：
  • 情感安抚 + 理性分析 + 具体操作建议
```

## 三、Agent 1：用户行为心智分析

### 3.1 核心职责

感知、建模、追踪用户的心理状态，输出"用户心智画像"供其他Agent决策。

### 3.2 数据输入源

| 数据源 | 采集方式 | 更新频率 | 隐私级别 |
|--------|---------|---------|---------|
| **初始画像** | 智能问卷（自适应12-15题） | 一次性+季度复评 | 中 |
| **交易行为** | 交易记录、持仓变动、止损设置 | 实时 | 高 |
| **APP行为** | 登录频率、停留时长、页面浏览模式 | 实时 | 中 |
| **对话文本** | 与AI对话的语气、用词、情绪表达 | 每次交互 | 中 |
| **情绪日记** | 用户主动标注心态（可选功能） | 用户触发 | 低 |
| **外部信号** | 可穿戴数据（心率/睡眠，用户授权） | 可选 | 高 |

### 3.4 核心算法模块

**模块A：行为序列分析器**
- 输入：用户行为时间序列（登录、浏览、交易等）
- 模型：Transformer Encoder + 时序异常检测
- 输出：行为模式变化信号（正常/异常/告警）

**模块B：情绪推理引擎**
- 输入：对话文本 + 行为信号 + 市场环境
- 模型：FinBERT微调 + 多模态融合网络
- 输出：情绪状态向量 + 置信度

**模块C：认知偏差检测器**
- 输入：交易决策序列 + 持仓结构 + 市场数据
- 模型：规则引擎 + 统计检验 + ML分类器
- 输出：各类偏差强度评分 + 触发告警

**模块D：风险承受力动态校准**
- 输入：问卷数据 + 行为揭示偏好 + 当前情绪
- 模型：贝叶斯更新框架
- 输出：实时风险承受力评分（stated vs revealed vs current三层）

### 3.5 干预策略矩阵

| 用户状态 | 检测信号 | 干预级别 | 干预方式 |
|---------|---------|---------|---------|
| 极度恐慌 | anxiety>0.8, 频繁查看 | L1（立即） | 推送安抚 + 暂停交易建议 + 历史案例 |
| 冲动贪婪 | greed>0.8, 频繁加仓 | L1（立即） | 风险提示 + 止盈建议 + 冷静期 |
| 轻度焦虑 | anxiety 0.5-0.8 | L2（适时） | 定期心理报告 + 教育内容 |
| 从众追热 | herding>0.7, 热股占比高 | L2（适时） | 对标分析 + 分散化建议 |
| 过度自信 | overconfidence>0.8 | L3（渐进） | 历史决策复盘 + 概率教育 |
| 情绪平稳 | 各指标正常区间 | 无干预 | 正常策略执行 |

## 四、Agent 2：市场情绪与资讯洞察

### 4.1 核心职责

实时感知多市场的情绪温度、关键资讯事件，为策略Agent提供环境判断。

### 4.2 数据输入源

| 数据源 | 覆盖市场 | 更新频率 |
|--------|---------|---------|
| **行情数据** | 大A/美股/港股实时行情 | 秒级 |
| **社交媒体** | 雪球/微博/Twitter/StockTwits | 分钟级 |
| **新闻资讯** | 财经新闻、公告、研报 | 分钟级 |
| **资金流向** | 北向资金/ETF申赎/大宗交易 | 日内 |

| **宏观数据** | GDP/CPI/利率/PMI | 事件驱动 |
| **链上数据**（可选） | 加密市场巨鲸动向 | 实时 |

### 4.4 核心算法模块

**模块A：多源情绪聚合引擎**
- 社交媒体NLP情感分析（中文FinBERT + 英文RoBERTa）
- 量化情绪指标计算（类Fear&Greed，7维度加权）

- 事件传导链分析（A→B市场的溢出效应）

**模块C：趋势研判与拐点预警**
- 情绪极值检测（历史分位数比较）
- 情绪-价格背离信号（情绪先行指标）
- 制度转换检测（Hidden Markov Model）

## 五、Agent 3：投资组合策略

### 5.1 核心职责

基于用户心智画像 + 市场情绪环境，生成个性化、动态调整的投资组合策略。

### 5.2 架构：三市场子Agent

```

│           策略主Agent (Strategy Master)       │
│    • 全局资产配置                             │
│    • 跨市场再平衡                             │
│    • 风险预算分配                             │
│    • 心智约束整合                             │
└──────┬──────────────┬──────────────┬─────────┘
       │              │              │
       ▼              ▼              ▼
┌────────────┐ ┌────────────┐ ┌────────────┐
│  大A子Agent │ │ 美股子Agent │ │ 港股子Agent │
│            │ │            │ │            │
│ • 板块轮动 │ │ • 行业配置 │ │ • AH溢价  │
│ • 个股筛选 │ │ • ETF选择  │ │ • 南向资金 │
│ • 择时信号 │ │ • 期权对冲 │ │ • 中概联动 │
│ • 打新策略 │ │ • DCA策略  │ │ • 汇率对冲 │
└────────────┘ └────────────┘ └────────────┘
```

### 5.3 策略生成流程

```
输入：
  ├── 用户心智画像（Agent1输出）
  ├── 市场情绪报告（Agent2输出）
  └── 用户投资约束（资金量、期限、偏好）

处理流程：
  Step 1: 全局资产配置决策
    • 确定股债配比 → 受用户当前风险承受力(current)约束
    • 确定市场分配 → 受市场情绪环境影响
    • 确定现金比例 → 受用户焦虑度影响

  Step 2: 各市场子策略生成
    • 大A：板块配置 + 个股/ETF选择
    • 美股：行业权重 + ETF/个股组合
    • 港股：标的选择 + 汇率对冲决策

  Step 3: 心智约束校验
    • 检验策略是否超出用户当前心理承受范围
    • 检验波动率是否匹配情绪稳定性
    • 调整至用户"舒适区"内

  Step 4: 输出包装
    • 策略可视化
    • 决策理由说明（可解释性）
    • 风险提示
```

## 六、Orchestrator 编排层设计

### 6.1 核心职责

- 路由用户意图到正确的Agent
- 协调多Agent输出的融合与冲突仲裁
- 控制策略调整的节奏（防止过度交易）
- 管理干预时机和优先级

### 6.2 决策优先级规则

```
优先级从高到低：
P0: 用户安全（检测到极端情绪+高风险操作 → 立即干预）
P1: 风险控制（止损触发、黑天鹅事件 → 立即行动）
P2: 心智适配（情绪变化显著 → 策略微调）
P3: 市场机会（情绪拐点、事件驱动 → 建议调整）
P4: 定期维护（再平衡、复评 → 计划执行）
```

### 6.3 冲突仲裁机制

```
场景：Agent2检测到市场极度恐慌（建议加仓），但Agent1检测到用户焦虑极高（建议减仓）

仲裁规则：
1. 用户心理安全 > 投资收益最优 → 优先保护用户心理
2. 策略：不加仓（即使可能错过底部），适当减仓至用户舒适区
3. 同时推送：情绪安抚 + 市场分析（"恐慌中有机会，但当前不急"）
4. 设置后续跟踪：当用户焦虑回落至阈值下方，再触发加仓建议
```

## 八、数据架构与技术栈

### 8.1 技术选型

| 层级 | 技术选择 | 说明 |
|------|---------|------|
| **LLM推理** | GPT-4级别 / Claude / 国产大模型 | Agent对话与推理核心 |
| **情感分析** | FinBERT(中文微调) + RoBERTa(英文) | 多语言情绪识别 |
| **行为分析** | Transformer Encoder + 时序异常检测 | 用户行为建模 |
| **策略引擎** | 因子模型 + 强化学习(PPO) + 规则引擎 | 投资组合优化 |
| **数据管道** | Kafka + Flink | 实时数据流处理 |
| **存储** | PostgreSQL + Redis + ClickHouse | 关系/缓存/时序 |
| **Agent框架** | LangGraph / AutoGen / 自研 | 多Agent编排 |
| **前端** | React Native / Flutter | 跨平台移动端 |

### 8.2 数据流架构

```
实时数据流：
行情API ──→ Kafka ──→ Flink处理 ──→ Agent2(市场情绪)
用户行为 ──→ Kafka ──→ Flink处理 ──→ Agent1(心智分析)
                                         │
                                         ▼
                              Orchestrator ──→ Agent3(策略) ──→ 用户

批量数据流：
历史行情 ──→ ClickHouse ──→ 因子计算/回测
用户档案 ──→ PostgreSQL ──→ 心理模型训练
策略日志 ──→ ClickHouse ──→ 绩效评估/模型迭代
```

## 十、风险与合规

### 10.1 合规要求
- 持牌要求：基金投顾牌照/证券投资咨询牌照
- 信息安全：等保三级、个人信息保护法合规
- 适当性管理：策略推荐必须匹配用户风险等级
- 免责声明：AI建议非投资承诺，不保证收益

### 10.2 核心风险
| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 模型误判用户情绪 | 错误干预 | 人工复核机制 + 用户反馈闭环 |
| 策略失效（市场极端） | 用户亏损 | 严格止损 + 压力测试 + 免责说明 |
| 数据泄露 | 用户信任崩塌 | 加密存储 + 最小化采集 + 本地计算 |
| 过度依赖AI | 用户丧失独立判断 | 教育内容 + 决策主权保留 |
